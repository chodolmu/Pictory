# Pictory - Game Design Document

> 색을 잃은 이마젠들을 되살리는 컬러 플러드 퍼즐 게임

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 컬러 플러드 + 라인 파괴 퍼즐 |
| 플랫폼 | 모바일 (iOS / Android) |
| 엔진 | Godot 4.x |
| IP | 제2의 왕국 (니노쿠니) |
| 타겟 유저 | 캐주얼 퍼즐 유저, 제2의 왕국 팬 |
| 제작 원칙 | **AI-only 구현** — 외부 리소스(이미지, 스프라이트, 사운드)나 수작업 애니메이션 없이 AI가 코드만으로 구현 가능한 범위로 설계. 비주얼은 Godot 내장 기능(도형, 색상, 셰이더, Tween, 파티클)으로 처리. |

### 1.1 한줄 요약

7x7 그리드에서 컬러 큐에서 배정된 색으로 연결 그룹을 칠하고, 한 줄(행 또는 열)을 같은 색으로 완성하면 해당 라인과 연결된 동색 그룹이 파괴되는 컬러 플러드 + 라인 파괴 + 중력 퍼즐. 이마젠을 좋아하는 후냐가 색을 잃은 이마젠들을 색칠로 깨워나가는 스토리.

## 2. 핵심 메커니즘

### 2.1 그리드 구조

- **7x7 메인 그리드** + 상단 **7행 버퍼** (블록이 위에서 떨어짐).
- 각 셀에는 하나의 색상이 배정된다.
- 버퍼의 블록은 메인 그리드 위에 대기하며, 빈 칸이 생기면 중력에 의해 아래로 떨어진다.

### 2.2 연결 판정

- 상하좌우 4방향으로 **같은 색상**인 칸끼리 "연결 그룹"을 형성한다 (BFS 탐색).
- 대각선은 연결로 판정하지 않는다.

### 2.3 색칠 액션

1. **컬러 큐**의 현재 색을 확인한다.
2. 그리드의 **아무 셀**을 탭한다.
3. 탭한 셀의 현재 색과 큐의 현재 색이 **다를 경우**:
   - 탭한 셀 + 해당 셀과 BFS로 연결된 전체 그룹이 큐의 현재 색으로 리컬러된다.
   - **턴 1 소모**.
   - 큐가 1칸 진행한다 (다음 색이 현재 색이 됨).
4. 탭한 셀이 큐의 현재 색과 같은 경우: 아무 일도 일어나지 않는다 (턴 미소모, 큐 미진행).

### 2.4 라인 파괴 조건 (행 + 열)

- 어떤 **행(가로줄) 또는 열(세로줄) 전체**가 동일한 색상이 되면 해당 라인이 **파괴 대상**으로 지정된다.
- 파괴 시, 해당 라인의 셀들 + 해당 라인과 **같은 색으로 연결된(BFS) 모든 그룹**이 함께 파괴된다.
- 한 번의 색칠 액션으로 복수의 행/열이 동시에 완성될 수 있다.

### 2.5 중력 및 블록 보충

- 라인 파괴 후, 빈 칸 위에 있는 블록들이 **중력에 의해 아래로 낙하**한다.
- 낙하 후 빈 칸은 버퍼에서 새로운 랜덤 블록으로 채워진다.
- 버퍼가 비면 새로운 랜덤 블록이 버퍼에 생성된다.

### 2.6 연쇄 콤보 (체인)

- 중력 낙하 및 블록 보충 후, 새로운 행/열이 자동으로 완성되면 **연쇄 파괴**가 발생한다.
- 연쇄는 더 이상 행/열이 완성되지 않을 때까지 계속된다.
- 연쇄 횟수에 따라 보너스 점수/효과를 부여할 수 있다.

### 2.7 클리어 조건

- **스토리 모드**: 주어진 턴 내에 목표 블록 수 이상을 파괴하면 클리어.
- **인피니티 모드**: 제한 시간 내 최대한 많은 블록을 파괴.

### 2.8 게임 오버 조건

- **스토리 모드**: 보유 턴이 0이 되었는데 목표 블록 수에 도달하지 못하면 게임 오버.
- **인피니티 모드**: 제한 시간이 0이 되면 종료.

## 3. 컬러 큐 시스템 (버퍼 블록 생성)

### 3.1 개요

버퍼에 채워지는 블록의 색상은 순수 랜덤이 아니라, **스트라이드 스케줄링(변형) + 마이크로 랜덤 오프셋**으로 생성된다. 순수 랜덤은 특정 색이 과도하게 몰리거나 부족해져 불공정하게 느껴지므로, 각 색이 대략 균등하게 출현하도록 보장한다.

### 3.2 동작 원리

- 사용 가능한 색상 수를 K라 하면, K개의 색이 한 사이클 내에 각각 1번씩 출현하도록 스트라이드를 구성한다.
- 사이클 내 순서에 **마이크로 랜덤 오프셋**을 적용하여 완전한 패턴 예측을 방지한다.
- 예: 4색 사용 시, 4개 블록마다 각 색이 대략 1번씩 등장하되 순서는 매번 약간 달라진다.

### 3.3 밸런스 파라미터 (Godot Inspector 노출)

| 파라미터 | 설명 | 기본값 (예시) |
|----------|------|--------------|
| `stride_length` | 한 사이클의 블록 수 (보통 색상 수와 동일) | 색상 수 |
| `random_offset_range` | 사이클 내 순서 교란 범위 (0이면 완전 균등, 클수록 랜덤에 가까움) | 1 |
| `color_count` | 사용 색상 수 | 스테이지별 설정 |

> 모든 파라미터는 Godot 에디터의 Inspector에서 직접 조정 가능하도록 `@export` 변수로 노출한다.

## 4. 게임 모드

### 4.1 스토리 모드

| 항목 | 내용 |
|------|------|
| 규모 | 10챕터 x 10스테이지 = 100스테이지 (확장 가능) |
| 해금 방식 | **챕터 단위 해금** — 챕터를 열 때만 재화가 필요. 챕터 내 10스테이지는 순서대로 플레이하되 개별 해금 비용 없음 |
| 진행 방식 | 턴 제한 내 목표 블록 수 파괴 |
| 턴 관리 | 스테이지별 리셋 |
| 기믹 | 챕터별로 신규 기믹 도입 |
| 연출 | 스테이지 시작 전 / 클리어 후 스토리 대화 파트 |
| 난이도 | 초반 스테이지는 쉽게 설계하여 자연스러운 튜토리얼 역할 |

#### 4.1.1 별 시스템 (스토리 모드 전용)

별은 클리어 시 **남은 턴 수**를 기준으로 평가한다 (최적해 비교가 아님).

| 조건 | 별 | 비고 |
|------|-----|------|
| 남은 턴 >= 3★ 임계값 | ★★★ | 퍼펙트 클리어 |
| 남은 턴 >= 2★ 임계값 | ★★ | 클리어 |
| 클리어 성공 (남은 턴 > 0) | ★ | 클리어 |
| 턴 소진, 목표 미달 | 없음 | 게임 오버 |

- 각 스테이지의 3★/2★ 임계값은 **레벨 디자인 단계에서 스테이지별로 개별 설정**한다.
- 인간 디자이너가 플레이 테스트를 통해 조정하는 값이며, 자동 산출이 아닌 수동 밸런싱이다.
- 예시: 15턴 제공, 목표 100블록 파괴인 스테이지 → 3★ = 남은 턴 5 이상, 2★ = 남은 턴 3 이상, 1★ = 클리어만 하면 획득.

### 4.2 인피니티 모드

| 항목 | 내용 |
|------|------|
| 해금 조건 | 없음 (처음부터 플레이 가능) |
| 진행 방식 | 제한 시간 내 최대한 많은 블록을 파괴 |
| 종료 조건 | 시간 소진 |
| 기믹 | 등장 가능 (시간 경과에 따라 점진적 도입) |
| 보상 | 점수 → 재화 변환 |

#### 4.2.1 보너스 시간 시스템

- 한 번에 **n개 이상**의 블록을 동시에 파괴하면, 파괴된 블록 수에 비례하여 **보너스 시간**이 추가된다.
- 연쇄 콤보로 대량 파괴 시 큰 보너스 시간을 획득할 수 있다.

| 파라미터 | 설명 | 기본값 (예시) |
|----------|------|--------------|
| `bonus_threshold_n` | 보너스 시간이 발동되는 최소 동시 파괴 블록 수 | 10 |
| `time_per_block` | 임계값 초과 블록당 추가되는 시간 (초) | 0.5 |

> 위 파라미터는 Godot Inspector에서 밸런스 조정 가능하도록 `@export` 노출한다.

#### 4.2.2 난이도 스케일링

시간이 경과할수록 아래 변수들이 조합되어 난이도가 상승한다:

- 색상 수 증가
- 기믹 등장 빈도 및 복합도 증가
- 버퍼 블록 생성 속도 변화

## 5. 기믹 시스템

### 5.1 개요

> **중요**: 기믹 시스템은 핵심 메커니즘이 "라인 파괴 + 중력" 방식으로 변경됨에 따라 **재설계가 필요**하다. 아래 구조는 챕터별 기믹 도입 방침을 유지하되, 구체적인 기믹 목록과 상호작용은 **TBD(검토 중)** 상태이다.

### 5.2 설계 방침

- **챕터별 신규 기믹 1개씩 도입**하는 구조는 유지한다 (챕터 1은 기믹 없음, 순수 퍼즐).
- 기존 컬러 통일 방식에서 설계된 기믹들(벽, 잠긴 칸, 얼음, 무지개, 폭탄, 타이머, 번짐, 거울 등)이 라인 파괴 + 중력 메커니즘에서 어떻게 작동할지 재검토가 필요하다. 확정된 기믹 후보 목록은 `gimmick-candidates.md` 참조.
- 일부 기믹은 그대로 적용 가능할 수 있고, 일부는 수정 또는 삭제, 새로운 기믹 추가가 필요할 수 있다.

| 2 | TBD | 검토 중 |

| 4 | TBD | 검토 중 |
| 5 | TBD | 검토 중 |
| 6 | TBD | 검토 중 |
| 7 | TBD | 검토 중 |
| 8 | TBD | 검토 중 |
| 9 | TBD | 검토 중 |
| 10 | **복합** | 기존 기믹들의 조합 (기믹 확정 후 결정) |

### 5.4 기믹 재설계 시 고려사항

- 중력 낙하와의 상호작용 (기믹 블록이 떨어질 때 어떻게 되는가?)
- 라인 파괴 시 기믹 블록의 처리 (파괴되는가? 잔류하는가?)
- 버퍼에서 기믹 블록이 생성될 수 있는가?
- 연쇄 콤보 시 기믹 발동 타이밍

> 기믹 상세 설계는 코어 메커니즘 프로토타입 완성 후 별도 논의에서 확정한다.

## 6. 점수 및 재화 시스템

### 6.1 점수

| 획득 조건 | 점수 |
|-----------|------|
| 블록 파괴 | 파괴된 블록 수 x 기본 점수 |
| 대량 파괴 보너스 | 한 번에 N개 이상 파괴 시 배율 적용 |
| 연쇄 콤보 보너스 | 연쇄 횟수에 따라 추가 배율 |
| 스테이지 클리어 (스토리) | 남은 턴 비례 클리어 보너스 |

### 6.2 재화 (단일 통화)

- **획득 경로**: 인피니티 모드 점수 → 재화 변환, 스토리 모드 클리어 보상
- **사용처**: 스토리 모드 챕터 해금 (챕터 단위로만 소모, 스테이지별 해금 비용 없음), 스킨, 힌트 아이템 등
- 향후 확장: 이벤트/시즌 상점 등

## 7. 동료 이마젠 시스템

### 7.1 개요

스토리 모드에서 이마젠의 색을 되찾아 깨우면, 해당 이마젠이 **동료**로 합류한다. 동료 이마젠은 퍼즐 플레이 중 사용할 수 있는 **고유 스킬**을 보유한다.

### 7.2 동료 획득

- 스토리 모드 특정 스테이지 클리어 시 해당 이마젠이 동료로 합류
- 챕터별 1~2마리의 동료 이마젠 획득 가능
- 획득 시 전용 연출 (이마젠에 색이 돌아오며 깨어나는 컷씬)

### 7.3 스킬 시스템

동료 이마젠의 스킬은 퍼즐 플레이 중 전략적으로 사용할 수 있는 특수 능력이다.

| 스킬 타입 | 예시 | 설명 |
|-----------|------|------|
| **색 변환** | 단색 폭풍 | 지정 색상의 칸을 모두 다른 색으로 변환 |
| **턴 회복** | 시간의 숨결 | 보너스 턴 +N 획득 |
| **정보 공개** | 미래의 눈 | 힌트 표시 |
| **범위 효과** | 무지개 파동 | 지정 영역 내 칸을 모두 같은 색으로 통일 |
| **기믹 해제** | 얼음 녹이기 | 특정 기믹을 즉시 해제 |
| **보호** | 색의 방패 | N턴간 기믹 효과 무효화 |

### 7.4 스킬 사용 규칙

- 스테이지 시작 전 동료 이마젠 1~2마리 **편성** (슬롯 제한)
- 스킬 사용 시 **쿨타임** 발생 (N턴 후 재사용 가능)
- 스킬 사용은 **턴을 소모하지 않음**
- 스토리 모드 / 인피니티 모드 모두 사용 가능

### 7.5 이마젠 성장 (향후 확장)

- 동일 이마젠 중복 획득 시 스킬 강화 (쿨타임 감소, 효과 범위 증가 등)
- 이마젠 레벨업 시스템 (재화 투자)

## 8. 컬렉션 시스템

### 8.1 개요

스토리 모드와 인피니티 모드를 진행하며 다양한 수집 요소를 모을 수 있다. 컬렉션은 플레이어의 장기적인 목표와 리텐션을 제공한다.

### 8.2 컬렉션 탭 구성

| 탭 | 설명 |
|----|------|
| **후냐 커스터마이징** | 후냐 외형 꾸미기. 획득한 커스터마이징 아이템으로 후냐의 외형을 변경 |
| **이마젠 동료 편성** | 보유 이마젠 관리 및 편성. 이마젠 정보 열람, 스킬 확인, 퍼즐 출전 편성 |
| **플레이어 아이콘** | 아이콘 선택. 획득한 아이콘 중 프로필에 표시할 아이콘을 선택 |

### 8.3 컬렉션 UI

## 9. 플레이어 프로필 시스템

### 9.1 개요

플레이어의 정체성을 나타내는 프로필 시스템. 메인 화면 좌측 상단에 닉네임과 아이콘이 표시된다.

### 9.2 구성 요소

| 항목 | 설명 |
|------|------|
| **닉네임** | 플레이어가 직접 설정하는 이름. 최초 접속 시 닉네임 입력 화면이 표시됨 |
| **플레이어 아이콘** | 컬렉션에서 획득한 아이콘 중 선택하여 표시. 기본 아이콘 1종 제공 |

### 9.3 닉네임 입력

- **최초 접속 시**: 게임 첫 실행 시 닉네임 입력 화면이 표시된다 (타이틀 화면 이후, 메인 화면 진입 전)
- **변경**: 옵션(설정) 화면에서 닉네임 변경 가능

### 9.4 프로필 표시 위치

- **메인 화면**: 좌측 상단에 아이콘 + 닉네임 상시 표시

## 11. 스토리

### 11.1 세계관

- **IP**: 제2의 왕국 (니노쿠니)
- **배경**: 이마젠들이 살아가는 세계

### 11.2 줄거리

이마젠을 누구보다 좋아하는 후냐. 어느 날 갑자기 모든 이마젠들이 색을 잃고 회색으로 변해 가만히 멈춰버린다. 후냐는 이마젠들에게 색을 되돌려주며 하나씩 깨워나간다. 그 과정에서 이마젠들이 색을 잃은 진짜 이유를 쫓아간다.

### 11.3 내러티브 구조

| 챕터 | 키워드 | 스토리 개요 |
|------|--------|------------|
| 1 | 시작 | 후냐가 이마젠들의 이상을 발견하고 색칠을 시작 |
| 2~3 | 탐색 | 주변 이마젠들을 깨우며 단서를 모음 |
| 4~6 | 전개 | 색을 잃은 원인에 대한 실마리 발견, 갈등 심화 |
| 7~9 | 클라이맥스 | 원인의 핵심에 접근, 난관 극복 |
| 10 | 결말 | 진실 규명, 이마젠들의 색 완전 복구 |

### 11.4 연출 방식

- 각 스테이지 **시작 전**: 대화 씬 (캐릭터 일러스트 + 텍스트)
- 각 스테이지 **클리어 후**: 대화 씬 또는 컷씬
- 챕터 시작/종료: 챕터 타이틀 연출
- 퍼즐 클리어 = 이마젠에게 색을 되돌려주는 행위 (게임플레이-내러티브 일체)

## 13. 기술 구현 방식

### 13.1 그리드 시스템

- **데이터 구조**: 7x7 메인 그리드 + 7행 버퍼 = 2D 배열 기반.
- 각 셀은 `{color, gimmick, gimmick_param}` 속성 보유.
- **연결 그룹 탐색**: BFS로 인접 동색 칸 탐색.

```
셀 데이터 구조:
{
  color: int          # 색상 인덱스 (0~K-1)
  gimmick: enum       # NONE, ... (기믹 확정 후 정의)
  gimmick_param: int  # 기믹 파라미터
}
```

### 13.2 핵심 로직 (의사코드)

```
func apply_color(touched_cell):
    new_color = color_queue.current()  # 큐의 현재 색
    if touched_cell.color == new_color:
        return  # 같은 색이면 무시 (큐 미진행)

    # 1. 연결 그룹 탐색 (BFS)
    group = find_connected_group(touched_cell)

    # 2. 색상 변경 (리컬러)
    for cell in group:
        cell.color = new_color

    # 3. 턴 소모
    turns -= 1

    # 4. 큐 진행
    advance_color_queue()

    # 5. 라인 파괴 판정 + 연쇄 처리
    process_line_destruction()

func process_line_destruction():
    loop:
        completed_lines = find_completed_lines()  # 행/열 모두 탐색
        if completed_lines is empty:
            break

        # 완성된 라인(행/열) + 연결된 동색 그룹 파괴
        cells_to_destroy = set()
        for line in completed_lines:
            line_color = get_line_color(line)
            for cell in line:
                connected = find_connected_group(cell)
                cells_to_destroy.add_all(connected)

        destroy_cells(cells_to_destroy)
        destroyed_count += cells_to_destroy.size()

        # 중력 적용 (블록 낙하)
        apply_gravity()

        # 버퍼에서 블록 보충
        fill_from_buffer()

    # 클리어/게임오버 판정
    check_clear_or_gameover()
```

### 13.3 솔버 (재설계 필요)

> **주의**: 기존 솔버는 "최소 턴으로 전체 색상 통일"을 목표로 한 BFS 완전탐색이었으나, 새로운 메커니즘(라인 파괴 + 중력 + 블록 보충)에서는 목적과 상태 공간이 완전히 달라진다.

- 새 메커니즘에서 솔버의 역할: 주어진 턴 내 목표 블록 수 파괴가 가능한지 검증
- 중력 + 랜덤 블록 보충으로 인해 비결정적 요소가 존재 → 완전 탐색이 아닌 확률적/시뮬레이션 기반 접근 필요
- 솔버 재설계는 코어 메커니즘 확정 후 진행

### 13.4 밸런싱 시뮬레이터

| 항목 | 내용 |
|------|------|
| 입력 | 레벨 데이터 + 시뮬레이션 횟수 + AI 전략 선택 |
| AI 전략 | **그리디**: 매 턴 가장 많은 라인(행/열) 완성을 노리는 선택 / **랜덤**: 무작위 선택 |
| 출력 지표 | 클리어율, 평균 파괴 블록 수, 최소/최대 턴 소모, 표준편차, 난이도 등급 자동 산정 |
| 활용 | 스토리 모드 별 임계값 적정성 검증, 인피니티 모드 난이도 곡선 조정 |

### 13.5 레벨 에디터

- **기능**:
  - 그리드 초기 색상 배치 설정
  - 기믹 배치 (확정 후)
  - 턴 수 설정
  - 목표 파괴 블록 수 설정
  - 별 임계값 설정 (3★/2★ 남은 턴 기준)
  - 사용 색상 수 설정
  - 프리뷰 플레이
  - 시뮬레이터 실행 → 난이도 확인
- **출력**: JSON 형식의 레벨 데이터

### 13.6 레벨 자동 생성기

- **입력 파라미터**: 색상 수, 목표 난이도, 목표 파괴 블록 수, 허용 기믹 목록
- **생성 프로세스**:
  1. 랜덤 색상 배치
  2. 기믹 랜덤 배치 (해당하는 경우)
  3. 시뮬레이터로 난이도 검증
  4. 목표 난이도 범위에 맞는지 확인
  5. 미달/초과 시 재생성
- **용도**: 인피니티 모드 스테이지 생성, 레벨 디자이너 초안 제공
- **주의**: 새 메커니즘에 맞게 재설계 필요

## 15. 리스크 및 고려사항

| 리스크 | 대응 |
|--------|------|
| 새 메커니즘의 재미 검증 부족 | Phase 1 프로토타입 단계에서 빠르게 핵심 루프 검증. 재미없으면 파라미터 조정 |
| 기믹 재설계 지연 | 코어 메커니즘 우선 완성 후 기믹 논의. 기믹 없이도 기본 재미가 보장되어야 함 |
| 컬러 큐 밸런싱 | 스트라이드 파라미터를 Inspector에 노출하여 빠른 튜닝 가능. 시뮬레이터로 검증 |
| 연쇄 콤보의 운 의존도 | 버퍼 블록이 랜덤이므로 연쇄가 운에 좌우될 수 있음. 컬러 큐로 완화하되, 과도한 운 의존 시 메커니즘 조정 |
| 솔버 재설계 난이도 | 비결정적 요소(랜덤 버퍼)로 인해 완전 탐색 불가. 시뮬레이션 기반 접근 필요 |
| 재화 그라인딩 피로도 | 시뮬레이터로 평균 획득량 검증, 적정 해금 비용 설정 |
| 제2의 왕국 IP 저작권 | IP 사용 범위 및 라이선스 확인 필요 |
| 이마젠 스킬 밸런싱 | 스킬이 퍼즐 난이도를 무력화하지 않도록 쿨타임/효과 범위 조절 |
| 인피니티 모드 보너스 시간 밸런싱 | 너무 쉬우면 무한 플레이, 너무 어려우면 짧은 플레이. 파라미터 튜닝 필수 |
