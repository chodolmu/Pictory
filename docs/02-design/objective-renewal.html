A. BFS Recolor에서 파생되는 목표

Color Domination — 색 지배

BFS Easy Impl P1

Rule

  • 보드의 X% 이상을 지정 색으로 만들기
  • 파괴가 아닌 "칠하기"가 목표
  • 라인이 완성되면 파괴되어 진행도가 리셋될 수 있음 → 긴장감
  • 파괴를 피하면서 칠해야 할 수도 있는 역설적 상황

JSON Schema

{ "goal": { "type": "color_domination", "target_color": 1, "target_percent": 70 } }

Example: "보드 70%를 파란색으로"

🔵 Blue 12/18 (67%)
📀 Turns 4
🤔 2행이 거의 파란색인데 완성되면 파괴됨!
💡 일부러 행 완성을 피하면서 칠해야 하는 딜레마
✓ 파란 15/25칸 이상 달성 시 클리어

Paint All — 전체 칠하기

BFS Easy Impl P2

Rule

  • 모든 활성 셀을 최소 1번 이상 BFS 리컬러
  • 각 셀에 "칠해짐" 플래그 — 한 번 칠해지면 체크
  • 구석이나 고립된 셀을 찾아서 칠하는 탐색 퍼즐
  • 돌(Stone) 뒤의 셀, 잠금(Locked) 셀이 난관

Big Flood — 대홍수

BFS Easy Impl P2

Rule

  • 한 번의 BFS로 N칸 이상 리컬러 달성
  • 작은 영역을 여러 번 칠하는 대신, 큰 영역을 만들어야 함
  • 미리 같은 색 영역을 넓게 준비하는 전략적 플레이
  • "N번 달성" 또는 "한 번이라도 달성" 변형 가능
{ "goal": { "type": "big_flood", "min_size": 12, "required_times": 1 } }

Example: "한 번에 12칸 이상 칠하기"

💡 빨간 영역이 파편화되어 있음 (3+3+2+...)
1 먼저 파란/초록을 빨간으로 칠해서 영역 연결
2 큰 빨간 영역을 다른 색으로 한 번에 BFS!
✓ 12칸 이상 달성 시 클리어

Target Lines — 지정 라인 파괴

Destroy Easy Impl P1

Rule

  • 특정 행 또는 열을 파괴해야 클리어
  • 하이라이트된 행/열이 목표로 표시됨
  • 아무 행이나 파괴하면 되는 게 아님 → 정밀한 색 배치 필요
  • 여러 지정 라인을 동시에 요구할 수 있음
{ "goal": { "type": "target_lines", "targets": [ {"axis": "row", "index": 1}, {"axis": "row", "index": 3}, {"axis": "col", "index": 2} ] } }

Example: "2행, 4행, 3열을 파괴하라"

💡 금테 = 목표 라인. 2행/4행(가로) + 3열(세로)
1 2행을 파란색으로 통일 → 파괴! [1/3]
2 낙하 후 새 배치에서 4행 처리
✓ 3개 라인 모두 파괴 시 클리어

Chain Combo — 연쇄 달성

Destroy Easy Impl P2

Rule

  • 1액션에서 N연쇄 이상 달성
  • 라인 파괴 → 낙하 → 자연스럽게 새 라인 완성의 반복
  • 연쇄가 일어나도록 보드를 미리 세팅하는 전략
  • "3연쇄 1회" 또는 "2연쇄 3회" 등 변형
{ "goal": { "type": "chain_combo", "min_chain": 3, "required_times": 1 } }

Example: "3연쇄를 1번 달성하라"

1 4행의 초록을 빨강으로 → 4행 빨강 완성 → 파괴
2 낙하 → 3행(파란) 유지 → 파괴 (2연쇄)
3 낙하 → 1행(초록)이 내려오며 완성 → 파괴 (3연쇄!)
✓ 3연쇄 달성! 클리어
🔴

Color Destroy — 색 지정 파괴

Destroy Easy Impl P1

Rule

  • 특정 색 블록을 N개 이상 파괴
  • 아무 라인이나 터트리는 게 아니라, 목표 색이 포함된 라인을 노려야
  • "파란 블록 15개 파괴" — 파란색으로 라인을 만들어야 의미 있음
  • 다중 색 목표도 가능: "빨강 10개 + 초록 8개"
{ "goal": { "type": "color_destroy", "targets": [ {"color": 0, "count": 15}, {"color": 2, "count": 8} ] } }

Implementation Priority

구현 난이도 + 게임플레이 임팩트 기준 우선순위

P1 — Must Have (Core Variety)

기존 시스템 최소 수정으로 즉시 적용 가능. 목표 다양성의 기반.

  • Color Domination — Pictory만의 독자 목표
  • Target Lines — "아무 행"이 아닌 "지정 행" 파괴
  • Color Destroy — 색 지정 파괴로 큐 관리 전략
  • Break All Ice — 기믹 제거 목표의 표준
  • Collect Items — 수집 오브젝트 시스템
  • Drop Down — 낙하 활용 (가장 인기 있는 목표 유형)
  • Multi-Goal — 위 목표들을 조합하는 프레임워크

P2 — Should Have (Depth)

게임에 깊이를 더하는 도전적 목표. P1 구현 후 순차 적용.

  • Paint All — 탐색형 퍼즐 목표
  • Big Flood — 대규모 BFS 도전
  • Chain Combo — 연쇄 설계 목표
  • Defuse Bombs — 긴장감 특화 스테이지
  • Purify — 오염 기믹 활용 목표
  • Clear Zone — 영역 지정 파괴

P3 — Nice to Have (Spectacle)

구현 비용이 높지만 강한 인상을 주는 스페셜 스테이지.

  • Boss Stage — 챕터 보스전
  • Survival — 방어 모드
