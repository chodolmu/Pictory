## M1: Core Prototype

> **목표**: 한 화면에서 컬러 플러드 + 라인 파괴 + 중력 + 연쇄 콤보 한 판 플레이 가능
> **관련 FR**: FR-01 ~ FR-10

---

### S01: 그리드 기반 (Week 1)

> **스프린트 목표**: 그리드가 화면에 렌더링되고 셀을 터치하면 좌표가 인식된다

| # | 일감 | 설명 | 관련 FR | 산출물 |
|---|------|------|---------|--------|
| 1 | git 초기화 | repo init + .gitignore + 초기 커밋 | - | `.git/` |
| 2 | Cell 데이터 클래스 | `cell.gd` — 색상(int), 좌표(x,y), 활성 플래그, 기믹 슬롯(미사용 예약) | - | `scripts/core/cell.gd` |
| 3 | Grid 데이터 모델 | `grid.gd` — N×N + N행 버퍼, 셀 CRUD, 초기화 시 완성 라인 방지(`ensure_no_completed_lines`) | FR-01 | `scripts/core/grid.gd` |
| 4 | Grid 렌더링 씬 | `game.tscn` + GridView 노드, ColorRect 기반 셀 표시, 그리드 크기 파라미터화(5~7) | FR-01 | `scenes/game/game.tscn` |
| 5 | 셀 터치 입력 처리 | 터치/클릭 → 그리드 좌표 변환, 시그널 발행 | FR-02 | 입력 → 좌표 매핑 동작 |
| 6 | 버퍼 영역 렌더링 | 메인 그리드 위 버퍼 행 시각화 (반투명/축소) | FR-01 | 버퍼 표시 |

**완료 기준**: 7×7 그리드 + 7행 버퍼가 화면에 보이고, 셀 터치 시 좌표가 콘솔에 출력된다

### S02: 코어 메카닉 (Week 2)

> **스프린트 목표**: 셀 터치 → BFS 리컬러 → 라인 파괴 → 중력 낙하가 한 사이클 동작한다

| # | 일감 | 설명 | 관련 FR | 산출물 |
|---|------|------|---------|--------|
| 1 | FloodFill — BFS 연결 탐색 | `flood_fill.gd` — 4방향 동색 그룹 탐색, 버퍼 포함 전체 범위 | FR-03 | `scripts/core/flood_fill.gd` |
| 2 | 리컬러 액션 | 터치한 셀의 BFS 그룹 전체를 지정 색으로 변경 | FR-04 | 리컬러 동작 |
| 3 | 같은 색 터치 무효 처리 | 현재 큐 색 == 셀 색이면 아무 동작 안 함 (턴 미소모, 큐 미진행) | FR-04 | 무효 터치 방어 |
| 4 | RowDestroy — 행 완성 판정 | `row_destroy.gd` — 메인 영역 행 검사, 완성 행의 셀 + BFS 연결 그룹 파괴 대상 산출 | FR-05 | `scripts/core/row_destroy.gd` |
| 5 | RowDestroy — 열 완성 판정 | 동일 로직을 열(세로)에 적용. 행+열 통합 destroy set 반환 | FR-05 | `row_destroy.gd` (행+열) |
| 6 | Gravity — 중력 낙하 | `gravity.gd` — 열별 아래→위 압축, 빈 칸을 버퍼에서 채움(랜덤 색) | FR-06 | `scripts/core/gravity.gd` |
| 7 | 파괴 → 중력 통합 테스트 | 리컬러 → 라인 판정 → 파괴 → 중력의 한 사이클 수동 검증 | FR-05,06 | 수동 플레이 테스트 |

**완료 기준**: 셀 터치 → 리컬러 → 행/열 파괴 → 중력 낙하가 한 번 동작한다 (연쇄 미포함)
