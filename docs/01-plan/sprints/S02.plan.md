## T1: FloodFill BFS (`flood_fill.gd`)

### 설명

주어진 시작 셀에서 4방향(상하좌우) BFS를 수행하여 같은 색상으로 연결된 셀 그룹을 찾는 유틸리티 클래스. **main grid + buffer 영역 전체**를 탐색 범위로 한다.

### Acceptance Criteria

- [ ] `flood_fill(grid: Grid, start_x: int, start_y: int) -> Array[Cell]` — 시작 셀과 같은 색상으로 연결된 모든 셀 반환
- [ ] 4방향 탐색 (상, 하, 좌, 우). 대각선 미포함
- [ ] main grid + buffer 영역 모두 탐색 (y 범위: `-grid_size` ~ `grid_size - 1`)
- [ ] 시작 셀의 color가 -1 (empty)이면 빈 배열 반환
- [ ] 결과에 시작 셀 자신도 포함
- [ ] 이미 방문한 셀 재방문 없음 (visited set)
- [ ] 단위 테스트: 3x3 grid에서 L자 모양 연결 그룹이 올바르게 검출

### Dependencies

- S01 완료 (Grid, Cell 클래스)

### Implementation Notes

**BFS 알고리즘**:
```
# flood_fill.gd
class_name FloodFill
extends RefCounted

static func flood_fill(grid: Grid, start_x: int, start_y: int) -> Array[Cell]:
    var start_cell = grid.get_cell(start_x, start_y)
    if start_cell == null or start_cell.color == -1:
        return []

    var target_color = start_cell.color
    var visited: Dictionary = {}  # Vector2i -> bool
    var result: Array[Cell] = []
    var queue: Array[Vector2i] = [Vector2i(start_x, start_y)]
    visited[Vector2i(start_x, start_y)] = true

    var directions = [
        Vector2i(0, -1),  # 상
        Vector2i(0, 1),   # 하
        Vector2i(-1, 0),  # 좌
        Vector2i(1, 0),   # 우
    ]

    while queue.size() > 0:
        var pos = queue.pop_front()
        var cell = grid.get_cell(pos.x, pos.y)
        if cell != null and cell.color == target_color:
            result.append(cell)
            for dir in directions:
                var next = pos + dir
                if not visited.has(next) and grid.is_valid_coord(next.x, next.y):
                    visited[next] = true
                    queue.append(next)

    return result
```

**핵심 설계 결정**:
- `static func`으로 구현하여 stateless 유틸리티로 사용
- Grid 객체를 매개변수로 받아 의존성 주입
- buffer 포함 탐색이 중요한 이유: RowDestroy에서 완성된 라인과 연결된 buffer 셀도 파괴 대상에 포함되기 때문

**Edge Cases**:
- 시작 셀이 grid 경계에 있는 경우 — `is_valid_coord` 체크로 처리
- grid 전체가 같은 색인 경우 — 전체 셀이 결과에 포함 (정상 동작)
- 1개 셀만 연결된 경우 (주변에 같은 색 없음) — 시작 셀 1개만 반환

**성능 고려**:
- 7x7 + 7 buffer = 98셀 최대이므로 성능 이슈 없음
- Dictionary 기반 visited가 Array 검색보다 빠름

### Output Files

- `scripts/core/flood_fill.gd`

## T2: Recolor Action

### 설명

플레이어가 셀을 터치하면, 해당 셀에서 BFS로 찾은 연결 그룹 전체를 **현재 활성 큐 색상**으로 변경(recolor)한다. S02에서는 ColorQueue가 아직 없으므로, 임시로 랜덤 색상 또는 고정 색상을 사용한다.

### Acceptance Criteria

- [ ] 셀 터치 → BFS 그룹 탐색 → 그룹 전체 색상 변경
- [ ] 변경 후 GridView가 즉시 시각적으로 갱신
- [ ] BFS 그룹이 1개(자기 자신뿐)여도 정상 recolor
- [ ] recolor 후 RowDestroy 검사 트리거 (T4 연결)
- [ ] 임시 색상 할당: `(current_color + 1) % num_colors` 또는 랜덤 — S03에서 ColorQueue로 교체
- [ ] input lock: recolor → destroy → gravity 완료까지 추가 입력 차단

### Dependencies

- T1 (FloodFill BFS)

### Implementation Notes

**Recolor 흐름** (GameManager에서 orchestration):
```
# game_manager.gd (S02 임시 로직)
func _on_cell_touched(x: int, y: int) -> void:
    if _processing:
        return
    _processing = true
    grid_view.lock_input()

    var cell = grid.get_cell(x, y)
    if cell == null or cell.color == -1:
        _processing = false
        grid_view.unlock_input()
        return

    var active_color = _get_next_color()  # 임시 구현
    # Same-color guard (T3)
    if cell.color == active_color:
        _processing = false
        grid_view.unlock_input()
        return

    # BFS recolor
    var group = FloodFill.flood_fill(grid, x, y)
    for c in group:
        grid.set_cell_color(c.x, c.y, active_color)

    grid_view.refresh()

    # Destroy + Gravity (T4~T6 연결)
    _do_destroy_gravity_cycle()

    _processing = false
    grid_view.unlock_input()
```

**임시 색상 할당** (S03 ColorQueue 도입 전):
```
var _temp_color_index: int = 0

func _get_next_color() -> int:
    _temp_color_index = (_temp_color_index + 1) % grid.num_colors
    return _temp_color_index
```

**GridView 갱신 전략**:
- recolor 직후 `grid_view.refresh()` 전체 갱신 호출
- 최적화 시 변경된 셀만 갱신: `for c in group: grid_view.refresh_cell(c.x, c.y)`
- S02에서는 전체 refresh 사용, 최적화는 후순위

### Output Files

- `scripts/game_manager.gd` (recolor 로직 추가)

## T3: Same-Color Touch Guard

### 설명

터치한 셀의 색상이 현재 활성 큐 색상과 동일하면 아무 동작도 하지 않는다. 턴을 소모하지 않고, 큐도 진행하지 않는다.

### Acceptance Criteria

- [ ] 터치 셀 color == active queue color → 아무 동작 없음
- [ ] 턴 카운터 변화 없음
- [ ] 큐 진행 없음
- [ ] input lock 걸리지 않음 (즉시 다시 터치 가능)
- [ ] 디버그: "Same color — no action" 로그 출력

### Dependencies

- T2 (Recolor action 구현 후)

### Implementation Notes

- T2의 recolor 흐름에서 이미 guard가 포함되어 있음 (`if cell.color == active_color: return`)
- 이 task는 해당 guard가 **확실히 올바르게 동작하는지** 검증하는 데 초점
- BFS 그룹의 색상이 아니라 **터치한 셀 1개의 색상**과 active color를 비교
- 향후 gimmick 셀은 이 guard를 bypass할 수 있으므로, guard를 별도 함수로 분리 권장:
  ```
  func _is_same_color_touch(cell: Cell, active_color: int) -> bool:
      return cell.color == active_color
  ```

### Output Files

- `scripts/game_manager.gd` (guard 로직 확인/보완)

## T4: RowDestroy Row Check (`row_destroy.gd`)

### 설명

main area의 모든 **행(row)**을 검사하여 완성된 행을 찾고, 해당 행의 셀 + BFS로 연결된 같은 색상 그룹을 합쳐 **파괴 대상 셀 집합(destroy set)**을 계산한다.

### Acceptance Criteria

- [ ] `check_rows(grid: Grid) -> Array[Cell]` — 파괴 대상 셀 배열 반환
- [ ] "완성된 행" = main area의 한 row(y값 동일)에서 모든 N개 셀이 같은 색상
- [ ] 완성된 행의 각 셀에서 BFS를 수행하여 연결된 같은 색 그룹을 destroy set에 추가
- [ ] 여러 행이 동시에 완성된 경우 모두 포함
- [ ] destroy set에 중복 셀 없음 (Set 기반 관리)
- [ ] buffer 영역 셀은 행 완성 판정 대상이 아니지만, BFS 연결 시 destroy set에 포함될 수 있음
- [ ] 완성된 행이 없으면 빈 배열 반환

### Dependencies

- T1 (FloodFill BFS — 연결 그룹 탐색용)

### Implementation Notes

**RowDestroy 클래스 구조**:
```
# row_destroy.gd
class_name RowDestroy
extends RefCounted

static func check_rows(grid: Grid) -> Array[Cell]:
    var destroy_set: Dictionary = {}  # Vector2i -> Cell (중복 방지)

    for y in range(grid.grid_size):  # main area rows only
        var row = grid.get_row(y)
        if _is_line_complete(row):
            _add_line_and_connected(grid, row, destroy_set)

    return destroy_set.values()

static func _is_line_complete(cells: Array[Cell]) -> bool:
    if cells.size() == 0:
        return false
    var first_color = cells[0].color
    if first_color == -1:
        return false
    for cell in cells:
        if cell.color != first_color:
            return false
    return true

static func _add_line_and_connected(grid: Grid, line_cells: Array[Cell], destroy_set: Dictionary) -> void:
    for cell in line_cells:
        var key = Vector2i(cell.x, cell.y)
        if not destroy_set.has(key):
            destroy_set[key] = cell
        # BFS로 연결된 같은 색 그룹 추가
        var connected = FloodFill.flood_fill(grid, cell.x, cell.y)
        for c in connected:
            var ckey = Vector2i(c.x, c.y)
            if not ckey in destroy_set:
                destroy_set[ckey] = c
```

**"행 완성 + BFS 연결" 의미**:
- 예시: row y=3이 모두 빨강이면, 해당 row 7셀이 파괴 대상
- 추가로 각 셀에서 BFS하면 row 외부에 있지만 빨강으로 연결된 셀도 파괴됨
- buffer 영역의 빨강 셀이 연결되어 있으면 buffer 셀도 파괴 → 이후 gravity에서 새 색상 충전

**핵심 주의사항**:
- BFS는 이미 모든 연결 셀을 찾으므로, 완성 행의 모든 셀에서 각각 BFS를 돌리면 중복이 많음
- 하지만 Dictionary 기반 destroy_set이 중복을 자동 제거하므로 정확성에는 문제 없음
- 최적화: 이미 destroy_set에 있는 셀에서는 BFS skip 가능 (성능 이슈 시)

**파괴 실행**:
- `destroy_set` 반환 후 호출자(GameManager)가 실제 파괴 수행:
  ```
  for cell in destroy_set:
      grid.set_cell_color(cell.x, cell.y, -1)  # -1 = empty
  ```

### Output Files

- `scripts/core/row_destroy.gd`

## T5: RowDestroy Column Check

### 설명

T4의 row 검사와 동일한 로직을 **열(column)**에 대해 수행하고, row destroy set과 통합하여 하나의 destroy set을 반환한다.

### Acceptance Criteria

- [ ] `check_columns(grid: Grid) -> Array[Cell]` — column 기반 파괴 대상 반환
- [ ] "완성된 열" = main area의 한 column(x값 동일)에서 모든 N개 셀이 같은 색상
- [ ] 완성 열의 각 셀에서 BFS → destroy set 추가 (row와 동일 로직)
- [ ] `check_all(grid: Grid) -> Array[Cell]` — row + column 통합 결과 (중복 제거)
- [ ] row와 column이 동시에 완성된 경우 (교차점 등) 정상 처리
- [ ] 단위 테스트: 십자(+) 패턴에서 row + column 동시 파괴 확인

### Dependencies

- T4 (Row check 완료 후 column 추가)

### Implementation Notes

**통합 API**:
```
# row_destroy.gd에 추가

static func check_columns(grid: Grid) -> Array[Cell]:
    var destroy_set: Dictionary = {}
    for x in range(grid.grid_size):
        var col = grid.get_column(x)  # main area cells only
        if _is_line_complete(col):
            _add_line_and_connected(grid, col, destroy_set)
    return destroy_set.values()

static func check_all(grid: Grid) -> Array[Cell]:
    var destroy_set: Dictionary = {}

    # Rows
    for y in range(grid.grid_size):
        var row = grid.get_row(y)
        if _is_line_complete(row):
            _add_line_and_connected(grid, row, destroy_set)

    # Columns
    for x in range(grid.grid_size):
        var col = grid.get_column(x)
        if _is_line_complete(col):
            _add_line_and_connected(grid, col, destroy_set)

    return destroy_set.values()

static func has_completed_lines(grid: Grid) -> bool:
    # chain combo용 빠른 체크 (S03에서 사용)
    for y in range(grid.grid_size):
        if _is_line_complete(grid.get_row(y)):
            return true
    for x in range(grid.grid_size):
        if _is_line_complete(grid.get_column(x)):
            return true
    return false
```

**동시 완성 시나리오**:
- Row y=3 전체 빨강 + Column x=4 전체 빨강 → 교차점 (4, 3)은 한 번만 포함
- Dictionary 기반이므로 자동 중복 제거
- BFS도 양쪽에서 동일 그룹을 찾지만 Dictionary가 처리

**`get_column()` 구현 주의**:
- Grid의 `get_column(x)` 는 **main area만** 반환해야 함 (y=0 ~ grid_size-1)
- Buffer 영역 포함 시 column 완성 판정이 깨짐
- BFS 확장 시에는 buffer까지 가지만, 라인 완성 판정은 main area만

### Output Files

- `scripts/core/row_destroy.gd` (기존 파일에 column 로직 추가)

## T6: Gravity (`gravity.gd`)

### 설명

파괴된 셀(empty, color = -1)을 아래로 채우는 gravity 시스템. 각 column에 대해 bottom-up으로 compaction하고, 최상단 빈 셀은 buffer에서 가져오거나 새 랜덤 색상으로 채운다.

### Acceptance Criteria

- [ ] `apply(grid: Grid) -> void` — gravity 적용 후 main area에 empty 셀 없음
- [ ] 각 column 독립적으로 처리 (column 간 간섭 없음)
- [ ] bottom-up compaction: 빈 셀 위의 셀이 아래로 내려옴
- [ ] buffer 셀도 gravity 대상: buffer의 셀이 main area로 내려올 수 있음
- [ ] 모든 셀이 내려온 후 남은 빈 자리는 랜덤 색상으로 채움
- [ ] gravity 후 buffer 영역도 다시 랜덤 색상으로 채워짐 (항상 buffer가 꽉 차 있어야 함)
- [ ] 셀 이동 시 `active` 플래그 갱신: buffer → main 이동 시 `active = true`

### Dependencies

- S01-T3 (Grid 데이터 모델)

### Implementation Notes

**Column Compaction 알고리즘**:
```
# gravity.gd
class_name Gravity
extends RefCounted

static func apply(grid: Grid) -> void:
    for x in range(grid.grid_size):
        _compact_column(grid, x)
    _refill_buffer(grid)

static func _compact_column(grid: Grid, x: int) -> void:
    # 전체 column 셀을 아래에서 위로 수집 (main + buffer)
    # y range: grid_size-1 (main bottom) → -grid_size (buffer top)
    var non_empty: Array[int] = []  # color values, bottom to top

    for y in range(grid.grid_size - 1, -grid.grid_size - 1, -1):
        var cell = grid.get_cell(x, y)
        if cell != null and cell.color != -1:
            non_empty.append(cell.color)

    # Bottom-up 재배치
    var write_y = grid.grid_size - 1  # main bottom
    for color in non_empty:
        grid.set_cell_color(x, write_y, color)
        if write_y >= 0:
            grid.get_cell(x, write_y).active = true
        write_y -= 1

    # 나머지 빈 자리 (-1로 마킹)
    while write_y >= -grid.grid_size:
        grid.set_cell_color(x, write_y, -1)
        write_y -= 1

static func _refill_buffer(grid: Grid) -> void:
    # buffer 영역의 빈 셀을 랜덤 색상으로 채움
    for y in range(-1, -grid.grid_size - 1, -1):
        for x in range(grid.grid_size):
            var cell = grid.get_cell(x, y)
            if cell != null and cell.color == -1:
                cell.color = randi() % grid.num_colors
                cell.active = false
```

**Gravity 동작 시각화 예시** (column x=2, grid_size=3):
```
Before:           After compact:     After refill:
buffer y=-3: [R]  buffer y=-3: [ ]   buffer y=-3: [?]
buffer y=-2: [ ]  buffer y=-2: [ ]   buffer y=-2: [?]
buffer y=-1: [B]  buffer y=-1: [ ]   buffer y=-1: [?]
---               ---                ---
main   y=0:  [ ]  main   y=0:  [B]   main   y=0:  [B]
main   y=1:  [G]  main   y=1:  [G]   main   y=1:  [G]
main   y=2:  [ ]  main   y=2:  [R]   main   y=2:  [R]
```

**핵심 주의사항**:
- `non_empty` 리스트를 bottom-up으로 수집하므로 상대적 순서가 유지됨
- buffer 셀이 main area로 이동 시 `active = true`로 변경 필수
- `_refill_buffer` 에서 랜덤 색상 사용 → S03에서 ColorQueue 기반으로 교체
- gravity 후 `ensure_no_completed_lines()` 는 호출하지 않음 — chain combo에서 처리

**S03 ColorQueue 연동 대비**:
- `_refill_buffer` 를 별도 함수로 분리하여 향후 ColorQueue에서 색상을 가져오도록 교체 가능
- 현재는 `randi() % num_colors` 사용

## T7: Destroy → Gravity Integration Test

### 설명

T2(Recolor) → T4/T5(RowDestroy) → T6(Gravity)를 한 사이클로 연결하여, 터치 → recolor → 라인 검사 → 파괴 → gravity가 정상 동작하는지 수동 테스트한다. Chain combo(반복 검사)는 S03 범위이므로 1회 cycle만 확인한다.

### Acceptance Criteria

- [ ] 터치 → recolor → 완성 라인 발견 → 파괴 → gravity → 화면 갱신 전체 흐름 작동
- [ ] 완성 라인이 없는 경우 파괴/gravity 건너뜀

**GameManager 1-cycle 파이프라인**:
```
# game_manager.gd
func _do_destroy_gravity_cycle() -> void:
    # 1. 완성 라인 검사
    var destroy_set = RowDestroy.check_all(grid)

    if destroy_set.size() == 0:
        print("No completed lines")
        return

    print("Destroyed cells: ", destroy_set.size())

    # 2. 파괴 (color → -1)
    for cell in destroy_set:
        grid.set_cell_color(cell.x, cell.y, -1)

    grid_view.refresh()
    # 선택: 파괴 시각 효과를 위해 await get_tree().create_timer(0.3).timeout

    # 3. Gravity
    Gravity.apply(grid)

    grid_view.refresh()
    # 선택: gravity 시각 효과를 위해 await get_tree().create_timer(0.2).timeout

    print("Cycle complete")
```

**수동 테스트 시나리오**:
1. grid 초기화 후, 거의 완성된 row를 수동 설정 (디버그 함수)
2. 남은 1칸을 recolor로 채워 완성
3. 파괴 + gravity 확인
4. column 동시 완성 시나리오도 테스트

**디버그 헬퍼 함수** (테스트용):

func _debug_set_row(y: int, color: int) -> void:
