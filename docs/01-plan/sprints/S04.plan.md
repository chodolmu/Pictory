## Task 1: 레벨 JSON 스키마 정의

### 설명

스토리 모드 스테이지의 구성 데이터를 정의하는 JSON 스키마를 작성한다. 이 스키마는 모든 레벨 JSON 파일의 기준이 되며, LevelLoader가 파싱 시 유효성 검증에 사용한다. 기믹 배치 필드는 S06에서 사용하므로 예약만 해 둔다.

### 스키마 필드

```json
{
  "stage_id": "ch01_s01",
  "chapter": 1,
  "stage_number": 1,
  "grid_size": 5,
  "num_colors": 3,
  "turn_limit": 15,
  "goal": {
    "type": "destroy_blocks",
    "target_count": 20
  },
  "star_thresholds": [3, 6, 10],
  "color_queue_config": {
    "stride": 3,
    "random_offset": 1
  },
  "gimmick_placements": [],
  "buffer_rows": 5,
  "metadata": {
    "name_ko": "첫 번째 퍼즐",
    "story_preview_key": "ch01_s01_preview"
  }
}
```

- `star_thresholds`: 남은 턴 기준 [1성, 2성, 3성] — `remaining_turns >= threshold[i]`이면 해당 별 획득
- `goal.type`: 현재 `destroy_blocks`만 지원, 향후 확장 가능
- `gimmick_placements`: `[{ "type": "locked", "x": 2, "y": 3 }]` 형태 (S06 구현)
- `buffer_rows`: 기본값은 `grid_size`와 동일, 명시적으로 오버라이드 가능

### Acceptance Criteria

- [ ] `resources/levels/schema.json` 파일이 존재하고, 위 필드를 모두 포함한다
- [ ] 필수 필드 누락 시 LevelLoader에서 에러를 출력할 수 있도록 required 목록이 명시되어 있다
- [ ] `gimmick_placements`가 빈 배열이어도 정상 파싱된다

### Dependencies

- 없음 (스키마 정의만)

### 구현 노트

- Godot에서는 JSON Schema validator가 내장되어 있지 않으므로, 스키마 파일은 문서/참조 용도로 작성하고, 실제 검증은 `level_loader.gd`에서 수동으로 수행한다
- 파일 경로: `resources/levels/schema.json`

### 파일

| 파일 | 타입 | 설명 |
|------|------|------|
| `resources/levels/schema.json` | JSON | 레벨 데이터 스키마 정의 |

## Task 2: LevelLoader (`level_loader.gd`)

### 설명

JSON 레벨 파일을 읽어 GDScript에서 사용 가능한 스테이지 설정 오브젝트로 변환하는 Autoload 싱글턴을 구현한다. `StageConfig` 리소스 클래스를 함께 정의한다.

### 상세 구현

**StageConfig 클래스** (`scripts/data/stage_config.gd`):

```gdscript
class_name StageConfig
extends RefCounted

var stage_id: String
var chapter: int
var stage_number: int
var grid_size: int
var num_colors: int
var turn_limit: int
var goal_type: String
var goal_target_count: int
var star_thresholds: Array[int]  # [1star, 2star, 3star]
var color_queue_stride: int
var color_queue_random_offset: int
var gimmick_placements: Array[Dictionary]
var buffer_rows: int
var metadata: Dictionary
```

**LevelLoader** (`scripts/data/level_loader.gd`):

```gdscript
extends Node  # Autoload singleton

func load_stage(stage_id: String) -> StageConfig
func load_chapter_stages(chapter: int) -> Array[StageConfig]
func _parse_json(json_data: Dictionary) -> StageConfig
func _validate_required_fields(json_data: Dictionary) -> bool
```

- `load_stage()`: `resources/levels/story/chapter_XX/stageYY.json` 경로 규칙으로 파일 로드
- `load_chapter_stages()`: 챕터 디렉토리 내 모든 스테이지 로드, `stage_number` 기준 정렬
- `_validate_required_fields()`: 필수 필드(`stage_id`, `grid_size`, `num_colors`, `turn_limit`, `goal`, `star_thresholds`) 존재 확인, 누락 시 `push_error()` + `null` 반환

### Acceptance Criteria

- [ ] `LevelLoader.load_stage("ch01_s01")`이 유효한 `StageConfig` 오브젝트를 반환한다
- [ ] `LevelLoader.load_chapter_stages(1)`이 10개의 `StageConfig` 배열을 반환한다
- [ ] 필수 필드 누락 시 에러 메시지를 출력하고 `null`을 반환한다
- [ ] 존재하지 않는 스테이지 ID 요청 시 에러 없이 `null`을 반환한다
- [ ] `gimmick_placements`가 빈 배열이면 빈 배열로 정상 처리된다

### Dependencies

- Task 1 (JSON 스키마)

### 구현 노트

- **Autoload 등록**: `Project > Project Settings > Autoload`에 `LevelLoader` 등록 (`scripts/data/level_loader.gd`)
- JSON 파싱: `JSON.parse_string()` 사용 (Godot 4.x)
- 파일 읽기: `FileAccess.open()` + `get_as_text()` → `JSON.parse_string()`
- 경로 패턴: `"res://resources/levels/story/chapter_%02d/stage_%02d.json" % [chapter, stage_number]`
- `StageConfig`는 `RefCounted` 확장하여 GC 자동 관리

### 파일

| 파일 | 타입 | 설명 |
|------|------|------|
| `scripts/data/level_loader.gd` | GDScript | Autoload — JSON 파싱 및 StageConfig 반환 |

## Task 4: StoryMode 로직 (`story_mode.gd`)

### 설명

스토리 모드의 게임 규칙을 관리하는 모듈을 구현한다. 턴 제한, N블록 파괴 목표, 별(star) 시스템을 포함한다. 기존 `game_manager.gd` 및 `turn_manager.gd`와 연동하여 모드별 규칙을 주입한다.

### 상세 구현

```gdscript
class_name StoryMode
extends Node

signal stage_cleared(stars: int, score: int, remaining_turns: int)
signal stage_failed()
signal turn_used(remaining: int)
signal blocks_destroyed(total: int, goal: int)

@export var config: StageConfig  # LevelLoader에서 주입

var remaining_turns: int
var destroyed_count: int
var goal_count: int
var is_active: bool

func initialize(stage_config: StageConfig) -> void
func on_action_performed(destroyed_blocks: int, combo_level: int) -> void
func _check_clear_condition() -> void
func _check_fail_condition() -> void
func _calculate_stars(remaining_turns: int) -> int
func get_progress() -> float  # destroyed_count / goal_count
func reset() -> void
```

**별 계산 로직**:
```
remaining_turns >= star_thresholds[2] → 3성
remaining_turns >= star_thresholds[1] → 2성
remaining_turns >= star_thresholds[0] → 1성
else → 0성 (클리어는 되었으나 별 없음)
```

**턴 흐름**:
1. 유효한 액션 수행 (같은 색 터치 제외) → `remaining_turns -= 1`
2. 파괴된 블록 수 누적 → `destroyed_count += destroyed_blocks`
3. `destroyed_count >= goal_count` → `stage_cleared` 시그널 (별 계산 포함)
4. `remaining_turns == 0` && `destroyed_count < goal_count` → `stage_failed` 시그널

### Acceptance Criteria

- [ ] `StoryMode.initialize(config)`로 스테이지 설정을 주입하면 턴/목표가 정상 세팅된다
- [ ] 유효 액션마다 `remaining_turns`가 1 감소한다
- [ ] 목표 블록 수 달성 시 `stage_cleared` 시그널이 올바른 별 수와 함께 발행된다
- [ ] 턴 소진 + 목표 미달 시 `stage_failed` 시그널이 발행된다
- [ ] 별 계산이 `star_thresholds` 기준으로 정확히 동작한다 (경계값 포함)
- [ ] `get_progress()`가 0.0~1.0 범위의 정규화된 진행도를 반환한다

### Dependencies

- Task 2 (LevelLoader, StageConfig)
- S03: `game_manager.gd`, `turn_manager.gd`

### 구현 노트

- `StoryMode`는 `GameManager`의 자식 노드로 추가하거나, `GameManager`가 현재 모드를 참조하는 구조로 설계
- **모드 전환 패턴**: `GameManager`에 `var current_mode: Node` 필드를 두고, 스토리/무한 모드 노드를 교체하는 Strategy 패턴 적용
- `game_manager.gd`의 기존 클리어/게임오버 판정 로직(S03 Task 7)을 `StoryMode`/`InfinityMode`로 위임하도록 리팩토링 필요
- `turn_manager.gd`의 턴 감소 로직은 `StoryMode`가 직접 호출하거나, 시그널로 연동
- 시그널 연결: `StoryMode.stage_cleared` → `GameManager` → 결과 팝업 표시
- `destroyed_count`는 연쇄 콤보 중 파괴된 블록도 모두 포함 (매 chain step마다 누적)

|------|------|------|
| `scripts/modes/story_mode.gd` | GDScript | 스토리 모드 규칙 관리 |

## Task 5: InfinityMode 로직 (`infinity_mode.gd`)

### 설명

무한 모드의 게임 규칙을 관리하는 모듈을 구현한다. 시간 제한 + 대량 파괴 시 보너스 시간 부여를 핵심 메커니즘으로 한다. 파괴 블록 수에 비례하여 시간이 추가되므로, 플레이어가 큰 연쇄를 만들수록 오래 플레이할 수 있다.

### 상세 구현

```gdscript
class_name InfinityMode
extends Node

signal time_updated(remaining: float, max_time: float)
signal bonus_time_added(amount: float, reason: String)
signal game_over(final_score: int, total_destroyed: int)

@export var initial_time: float = 60.0
@export var max_time: float = 90.0
@export var bonus_threshold: int = 5       # 보너스 발동 최소 파괴 수
@export var bonus_coefficient: float = 0.5  # 파괴 수당 보너스 시간(초)
@export var combo_time_multiplier: float = 1.2  # 콤보 레벨당 보너스 배율

var remaining_time: float
var total_destroyed: int
var total_score: int
var is_active: bool
var _timer_paused: bool  # 연쇄 중 일시정지용

func initialize() -> void
func _process(delta: float) -> void  # 타이머 감소
func on_action_performed(destroyed_blocks: int, combo_level: int) -> void
func _calculate_bonus_time(destroyed_blocks: int, combo_level: int) -> float
func pause_timer() -> void   # 연쇄 연출 중 호출
func resume_timer() -> void
func get_time_ratio() -> float  # remaining_time / max_time
```

**보너스 시간 계산**:
```
if destroyed_blocks >= bonus_threshold:
    bonus = (destroyed_blocks - bonus_threshold + 1) * bonus_coefficient * (combo_time_multiplier ^ combo_level)
    remaining_time = min(remaining_time + bonus, max_time)
```

- `bonus_threshold = 5`: 5개 이상 파괴 시부터 보너스 발동
- `bonus_coefficient = 0.5`: 초과 블록당 0.5초
- `combo_time_multiplier = 1.2`: 연쇄 2단계면 ×1.2, 3단계면 ×1.44 ...
- `max_time` 캡: 보너스가 아무리 많아도 90초를 초과하지 않음

**타이머 동작**:
- `_process(delta)`에서 `remaining_time -= delta` (연쇄 연출 중에는 `_timer_paused = true`로 정지)
- `remaining_time <= 0` → `game_over` 시그널 발행

### Acceptance Criteria

- [ ] 초기 시간(60초)이 정상 세팅되고 매 프레임 감소한다
- [ ] 5개 이상 블록 파괴 시 보너스 시간이 추가된다
- [ ] 보너스 시간이 `max_time`을 초과하지 않는다
- [ ] 연쇄 콤보 레벨에 따라 보너스 배율이 정확히 적용된다
- [ ] 4개 이하 파괴 시 보너스 시간이 추가되지 않는다
- [ ] 시간 0 도달 시 `game_over` 시그널이 정확한 점수/파괴 수와 함께 발행된다
- [ ] 연쇄 연출 중 타이머가 일시정지된다
- [ ] 모든 `@export` 파라미터가 Inspector에서 조절 가능하다

### Dependencies

- S03: `game_manager.gd`, `chain_combo.gd`

### 구현 노트

- `_process(delta)` 사용 — `Timer` 노드 대신 직접 delta 감소하여 pause/resume 세밀 제어
- `chain_combo.gd`의 연쇄 시작/종료 시그널에 `pause_timer()`/`resume_timer()` 연결
- `@export` 변수로 Inspector에서 밸런싱 파라미터 조절 가능하게 설계
- `GameManager`에서 `StoryMode`와 동일한 인터페이스(`on_action_performed`)를 호출하므로 Strategy 패턴 일관성 유지
- 하이스코어 저장은 S05의 `SaveManager`에서 처리

#### 난이도 스케일링 (GDD §4.2.2)

시간 경과에 따라 아래 파라미터를 단계적으로 변경하여 난이도를 올린다. 각 임계 시간(초)은 `@export`로 Inspector 노출.

```gdscript
@export var scaling_enabled: bool = true
@export var scaling_intervals: Array[float] = [60.0, 120.0, 180.0]  # 경과 시간 임계값
# 각 단계에서: 색상 수 증가, 기믹 등장 빈도 증가

var elapsed_time: float = 0.0
var current_scale_level: int = 0  # 0 = 초기, 1/2/3 = 단계별

func _process(delta: float) -> void:
    if not is_active: return
    elapsed_time += delta
    remaining_time -= delta
    _check_scaling()

func _check_scaling() -> void:
    if not scaling_enabled: return
    var new_level = 0
    for i in range(scaling_intervals.size()):
        if elapsed_time >= scaling_intervals[i]:
            new_level = i + 1
    if new_level > current_scale_level:
        current_scale_level = new_level
        emit_signal("difficulty_scaled", current_scale_level)

signal difficulty_scaled(level: int)
# GameManager가 이 시그널을 받아 ColorQueue의 색상 수, GimmickManager의 등장 빈도를 갱신
```

> **S04 범위**: 스케일링 시그널 발행까지 구현. 실제 색상 수 변경(ColorQueue 연동)은 S07(기믹 완성) 이후 연결.

## Task 6: 점수 시스템 (`economy.gd`)

### 설명

블록 파괴 시 점수 계산과 점수→재화 변환을 담당하는 Autoload 싱글턴을 구현한다. 스토리 모드와 무한 모드 모두에서 사용되는 공통 점수 시스템이다.

### 상세 구현

```gdscript
extends Node  # Autoload singleton

@export var base_point_per_block: int = 10
@export var combo_bonus_base: int = 50
@export var combo_bonus_multiplier: float = 1.5
@export var score_to_currency_ratio: float = 0.01  # 100점 = 1재화

signal score_changed(new_score: int)
signal currency_earned(amount: int)

var current_score: int = 0
var current_combo_level: int = 0

func reset() -> void
func calculate_action_score(destroyed_blocks: int, combo_level: int) -> int
func add_score(destroyed_blocks: int, combo_level: int) -> int  # 실제 점수 적용 + 시그널
func convert_score_to_currency(score: int) -> int
func get_total_currency_earned() -> int
```

**점수 계산 공식**:
```
action_score = destroyed_blocks * base_point_per_block
             + combo_bonus_base * (combo_bonus_multiplier ^ combo_level)
             (combo_level 0일 때 콤보 보너스 없음)

최종 재화 = floor(total_score * score_to_currency_ratio)
```

- `combo_level`: 0 = 기본, 1 = 1연쇄, 2 = 2연쇄 ...
- 콤보 보너스는 `combo_level >= 1`일 때만 적용
- 예시: 8블록 파괴 + 2연쇄 = 80 + 50 * 1.5^2 = 80 + 112.5 = 192점

### Acceptance Criteria

- [ ] `calculate_action_score(8, 2)`가 192를 반환한다 (소수점 내림)
- [ ] `add_score()` 호출 시 `score_changed` 시그널이 발행된다
- [ ] `convert_score_to_currency(1500)`이 15를 반환한다
- [ ] `combo_level == 0`이면 콤보 보너스가 0이다
- [ ] `reset()` 호출 시 `current_score`가 0으로 초기화된다

### Dependencies

- S03: `chain_combo.gd` (콤보 레벨 정보 제공)

### 구현 노트

- **Autoload 등록**: `Project > Project Settings > Autoload`에 `Economy` 등록
- 재화(`currency`)는 게임 전체에서 단일 재화만 사용 (추후 S11에서 상점 연동)
- 점수 계산은 `int`로 처리 (`floori()` 사용)
- `chain_combo.gd`에서 연쇄 단계마다 `Economy.add_score(destroyed, combo_level)` 호출
- 스토리 모드: 스테이지 종료 시 `convert_score_to_currency()` 호출하여 재화 획득
- 무한 모드: 게임오버 시 동일 처리
