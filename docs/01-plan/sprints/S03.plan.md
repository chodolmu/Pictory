## T1: ChainCombo (`chain_combo.gd`)

### 설명

destroy → gravity → recheck를 **완성 라인이 없을 때까지** 반복하는 chain combo 시스템. 무한 루프 방지를 위해 `MAX_CHAIN = 20` 제한을 둔다.

### Acceptance Criteria

- [ ] `execute(grid: Grid) -> ChainResult` — chain 실행 후 결과 반환
- [ ] `ChainResult`: `chain_count: int`, `total_destroyed: int`, `destroyed_per_chain: Array[int]`
- [ ] 매 chain step: RowDestroy.check_all → 파괴 → Gravity.apply → recheck
- [ ] 완성 라인 없으면 chain 종료
- [ ] `MAX_CHAIN = 20` 도달 시 강제 종료 + 경고 로그
- [ ] chain_count = 0 (첫 recolor 후 완성 라인 없음)도 정상 처리
- [ ] 디버그: 각 chain step마다 "Chain X: destroyed Y cells" 출력

### Dependencies

- S02 완료 (RowDestroy, Gravity)

### Implementation Notes

**ChainCombo 구조**:
```
# chain_combo.gd
class_name ChainCombo
extends RefCounted

const MAX_CHAIN: int = 20

class ChainResult:
    var chain_count: int = 0
    var total_destroyed: int = 0
    var destroyed_per_chain: Array[int] = []

static func execute(grid: Grid) -> ChainResult:
    var result = ChainResult.new()

    for i in range(MAX_CHAIN):
        var destroy_set = RowDestroy.check_all(grid)

        if destroy_set.size() == 0:
            break

        # 파괴
        for cell in destroy_set:
            grid.set_cell_color(cell.x, cell.y, -1)

        # Gravity
        Gravity.apply(grid)

        result.chain_count += 1
        result.total_destroyed += destroy_set.size()
        result.destroyed_per_chain.append(destroy_set.size())

        print("Chain ", result.chain_count, ": destroyed ", destroy_set.size(), " cells")

    if result.chain_count >= MAX_CHAIN:
        push_warning("ChainCombo: MAX_CHAIN reached!")

    return result
```

**Chain 시각화 타이밍** (GameManager에서 처리):
- 즉시 실행 버전 (S03 기본): chain 전체를 한 번에 실행 후 최종 상태만 렌더링
- 애니메이션 버전 (향후): chain step 사이에 `await` 삽입하여 파괴 → gravity 시각 효과
- 애니메이션 버전을 위해 `execute_async(grid, callback)` 패턴도 고려:
  ```
  # 향후 async 버전 스케치
  static func execute_async(grid: Grid, on_step: Callable) -> ChainResult:
      # ... 각 step 후 on_step.call(chain_index, destroy_set) 호출
  ```

**Gravity refill과 ColorQueue 연동**:
- S02에서 `Gravity._refill_buffer`가 `randi()` 사용
- S03에서 ColorQueue가 도입되면, refill 시 ColorQueue에서 색상을 가져오도록 변경 필요
- 방법: `Gravity.apply(grid, color_source: Callable)` 로 색상 제공자를 주입
  ```
  static func apply(grid: Grid, get_color: Callable = func(): return randi() % grid.num_colors) -> void:
  ```
- 또는 Grid에 `color_provider` 프로퍼티를 두어 간접 참조

**Edge Cases**:
- Chain 1에서 gravity 후 새로 채워진 셀이 우연히 라인을 완성 → Chain 2 발동 (정상)
- 대량 파괴로 거의 전체 grid가 비어 gravity 후 대부분 랜덤 → 라인 완성 확률 낮음
- MAX_CHAIN 도달은 극히 드물지만 안전장치 필수

### Output Files

- `scripts/core/chain_combo.gd`

## T2: ColorQueue Stride Scheduling (`color_queue.gd`)

### 설명

플레이어에게 할당되는 색상 순서를 결정하는 ColorQueue 시스템. stride scheduling + random offset 방식으로 색상을 생성하며, 현재 활성 색상 1개 + 다음 3개를 유지한다.

### Acceptance Criteria

- [ ] `@export var stride_length: int = 3` — stride 주기
- [ ] `@export var offset_range: int = 1` — 랜덤 오프셋 범위 (-offset_range ~ +offset_range)
- [ ] `@export var num_colors: int = 5` — 색상 수
- [ ] 큐 크기: 항상 4개 유지 (1 active + 3 next)
- [ ] `get_active_color() -> int` — 현재 활성 색상
- [ ] `get_next_colors() -> Array[int]` — 다음 3개 색상
- [ ] `advance() -> void` — 큐 진행 (active 소비, 새 색상 1개 생성하여 뒤에 추가)
- [ ] stride scheduling: 기본 순서는 `0, 1, 2, 3, 4, 0, 1, ...` 에 random offset 적용
- [ ] 같은 색이 연속으로 3번 이상 나오지 않음 (선택 제약)

### Dependencies

- 없음 (독립 구현 가능)

### Implementation Notes

**Stride Scheduling 알고리즘**:
```
# color_queue.gd
class_name ColorQueue
extends RefCounted

@export var stride_length: int = 3
@export var offset_range: int = 1
@export var num_colors: int = 5

var _queue: Array[int] = []
var _base_index: int = 0  # stride 기본 인덱스

func _init() -> void:
    _fill_queue()

func _fill_queue() -> void:
    while _queue.size() < 4:
        _queue.append(_generate_next())

func _generate_next() -> int:
    # stride 기본 색상
    var base_color = _base_index % num_colors

    # random offset 적용
    var offset = randi_range(-offset_range, offset_range)
    var color = (base_color + offset) % num_colors
    if color < 0:
        color += num_colors

    _base_index += 1

    # 연속 중복 방지 (선택사항)
    if _queue.size() >= 2 and _queue[-1] == color and _queue[-2] == color:
        color = (color + 1) % num_colors

    return color

func get_active_color() -> int:
    return _queue[0]

func get_next_colors() -> Array[int]:
    return _queue.slice(1, 4)

func advance() -> void:
    _queue.pop_front()
    _fill_queue()

func peek_all() -> Array[int]:
    return _queue.duplicate()
```

**Stride Scheduling 설명**:
- `stride_length`는 한 번에 같은 패턴이 반복되는 주기 (예: stride=3이면 3턴마다 비슷한 색상 패턴)
- `offset_range`로 약간의 랜덤성 부여 → 완전 랜덤도 아니고 완전 예측 가능하지도 않은 밸런스
- `_base_index`가 계속 증가하며 `% num_colors`로 순환

**HTML 프로토타입 매칭**:
- 프로토타입: 1 active + 3 next = 4개 큐
- 터치 성공(recolor 발생) 시에만 `advance()` 호출
- same-color touch guard에 걸리면 `advance()` 호출하지 않음

**Gravity refill 연동**:
- buffer refill 시 ColorQueue에서 색상을 가져올지, 별도 랜덤을 사용할지 결정 필요
- **권장**: buffer refill은 별도 랜덤 사용 (ColorQueue는 플레이어 액션 전용)
- 이유: buffer refill이 queue를 소모하면 queue 예측이 어려워져 플레이어 경험 저하

### Output Files

- `scripts/core/color_queue.gd`

### 설명

현재 활성 색상(큰 원) + 다음 3개 색상(작은 원)을 화면에 표시하는 UI 컴포넌트.

### Acceptance Criteria

- [ ] 다음 3개: 작은 원 (반지름 ~14px)으로 나란히 표시

- [ ] `advance()` 호출 시 큐가 한 칸씩 이동하는 것이 시각적으로 확인 가능

### Dependencies

- T2 (ColorQueue 데이터 모델)

### Implementation Notes

**UI 구현 방식 — `_draw()` 기반**:
```
# color_queue_ui.gd
class_name ColorQueueUI
extends Control

var _color_queue: ColorQueue

const ACTIVE_RADIUS: float = 24.0
const NEXT_RADIUS: float = 14.0
const SPACING: float = 12.0

func setup(queue: ColorQueue) -> void:
    _color_queue = queue
    queue_redraw()

func refresh() -> void:
    queue_redraw()

func _draw() -> void:
    if _color_queue == null:
        return

    var palette = GridView.COLOR_PALETTE  # 공유 팔레트 참조

    # Active color (큰 원)
    var active_color = _color_queue.get_active_color()
    var active_pos = Vector2(ACTIVE_RADIUS + 8, ACTIVE_RADIUS + 8)
    draw_circle(active_pos, ACTIVE_RADIUS, palette[active_color])
    draw_arc(active_pos, ACTIVE_RADIUS, 0, TAU, 32, Color.WHITE, 2.0)

    # Next colors (작은 원들)
    var next_colors = _color_queue.get_next_colors()
    var next_x = active_pos.x + ACTIVE_RADIUS + SPACING + NEXT_RADIUS
    for i in range(next_colors.size()):
        var pos = Vector2(next_x, active_pos.y)
        draw_circle(pos, NEXT_RADIUS, palette[next_colors[i]])
        draw_arc(pos, NEXT_RADIUS, 0, TAU, 24, Color(1, 1, 1, 0.6), 1.5)
        next_x += NEXT_RADIUS * 2 + SPACING * 0.5
```

**대안 — Node 기반** (ColorRect 원형은 불가, shader 필요):
- Godot 4.x에서 원형 표시: `_draw()` + `draw_circle()` 이 가장 간단
- 또는 Polygon2D, TextureRect + 원형 텍스처 (에셋 없이는 `_draw()` 권장)

**배치 위치**:
- grid 상단 중앙, buffer 위 영역
- 또는 grid 우측에 세로 배치
- `@export var layout_direction: String = "horizontal"` 로 설정 가능하게

**COLOR_PALETTE 공유**:
- `GridView.COLOR_PALETTE`를 autoload 또는 별도 const 파일로 분리 권장
- 임시: `color_queue_ui.gd`에서 GridView 참조 또는 동일 const 복사
- 정리: `scripts/core/constants.gd` (autoload)에 팔레트 정의 → 모든 UI에서 참조

## T4: Turn Manager (`turn_manager.gd`)

### 설명

게임 모드별 턴 관리 시스템. Story mode에서는 제한된 턴 내에 목표 달성, Infinity mode에서는 턴이 증가하며 계속 플레이.

### Acceptance Criteria

- [ ] `@export var mode: String = "story"` — "story" 또는 "infinity"
- [ ] `@export var max_turns: int = 30` — story mode 최대 턴 수
- [ ] `turns_used: int` — 사용된 턴 수
- [ ] `turns_remaining: int` — 남은 턴 수 (story mode)
- [ ] `use_turn() -> void` — 턴 1 소모 (유효한 recolor 액션 시에만 호출)
- [ ] `is_game_over() -> bool` — story: 턴 소진 && 목표 미달성, infinity: false (시간 기반은 별도)
- [ ] signal `turn_changed(turns_used: int, turns_remaining: int)` — HUD 갱신용
- [ ] story mode: 턴 감소 (-1), infinity mode: 턴 증가 (+1)

### Dependencies

- 없음 (독립 구현)

### Implementation Notes

```
# turn_manager.gd
class_name TurnManager
extends RefCounted

signal turn_changed(turns_used: int, turns_remaining: int)

@export var mode: String = "story"  # "story" | "infinity"
@export var max_turns: int = 30

var turns_used: int = 0

var turns_remaining: int:
    get:
        if mode == "story":
            return max_turns - turns_used
        return -1  # infinity mode에서는 의미 없음

func use_turn() -> void:
    turns_used += 1
    turn_changed.emit(turns_used, turns_remaining)

func is_game_over() -> bool:
    if mode == "story":
        return turns_remaining <= 0
    return false  # infinity mode는 별도 조건

func reset() -> void:
    turns_used = 0
    turn_changed.emit(turns_used, turns_remaining)
```

**턴 소모 시점**:
- `GameManager._on_cell_touched()` 에서 same-color guard 통과 후, recolor 실행 직전에 `use_turn()` 호출
- chain combo 동안에는 추가 턴 소모 없음 (1 touch = 1 turn)

**Infinity mode 확장**:
- 현재 시간 기반 종료는 T7에서 별도 타이머로 처리
- TurnManager는 턴 카운트만 담당

```
# hud.gd
class_name HUD
extends Control

@onready var stage_label: Label = $TopBar/StageLabel
@onready var turn_label: Label = $TopBar/TurnLabel
@onready var destroyed_label: Label = $BottomBar/DestroyedLabel
@onready var goal_label: Label = $BottomBar/GoalLabel
@onready var chain_label: Label = $ChainLabel

var _mode: String = "story"
var _total_destroyed: int = 0

func setup(mode: String, stage: int = 1, goal: int = 0, max_turns: int = 30) -> void:
    _mode = mode
    if mode == "story":
        stage_label.text = "Stage %d" % stage
        goal_label.text = "Goal: %d" % goal
        turn_label.text = "Turns: %d" % max_turns
    else:
        stage_label.text = "Infinity"
        goal_label.text = ""
        turn_label.text = "Turns: 0"

func update_turns(used: int, remaining: int) -> void:
    if _mode == "story":
        turn_label.text = "Turns: %d" % remaining
    else:
        turn_label.text = "Turns: %d" % used

func update_destroyed(count: int) -> void:
    _total_destroyed += count
    destroyed_label.text = "Destroyed: %d" % _total_destroyed

func show_chain(chain_count: int) -> void:
    if chain_count > 1:
        chain_label.text = "Chain x%d!" % chain_count
        chain_label.visible = true
        # 1.5초 후 숨김
        var tween = create_tween()
        tween.tween_property(chain_label, "modulate:a", 0.0, 0.8).set_delay(0.7)
        tween.tween_callback(func(): chain_label.visible = false; chain_label.modulate.a = 1.0)
    else:
        chain_label.visible = false
```

```
func _update_bar_color() -> void:
```

## T7: Clear/GameOver Check

### 설명

매 턴 종료 후 게임 클리어 또는 게임 오버 조건을 검사한다.

### Acceptance Criteria

- [ ] Story mode 클리어: `total_destroyed >= goal`
- [ ] Story mode 게임 오버: `turns_remaining == 0 && total_destroyed < goal`
- [ ] Infinity mode 종료: `timer <= 0`
- [ ] 클리어 시: "Stage Clear!" 텍스트 표시 + 입력 차단
- [ ] 게임 오버 시: "Game Over" 텍스트 표시 + 입력 차단
- [ ] chain combo 완료 후에 판정 (chain 중에는 판정하지 않음)
- [ ] signal `game_cleared` / `game_over` 발신 (향후 scene 전환용)

### Dependencies

- T4 (TurnManager), T5 (HUD)

### Implementation Notes

**판정 로직 위치**: GameManager에서 chain combo 완료 후 호출

### Implementation Notes

**GameManager 전체 구조**:
```
# game_manager.gd
class_name GameManager
extends Node2D

signal game_cleared
signal game_over

# 설정
@export var mode: String = "story"
@export var grid_size: int = 7
@export var num_colors: int = 5
@export var max_turns: int = 30
@export var goal: int = 100
@export var time_limit: float = 120.0  # infinity mode

# Core systems
var grid: Grid
var color_queue: ColorQueue
var turn_manager: TurnManager
var chain_combo: ChainCombo

# UI references
@onready var grid_view: GridView = $GridView
@onready var color_queue_ui: ColorQueueUI = $ColorQueueUI
@onready var hud: HUD = $HUD

# State
var _total_destroyed: int = 0
var _processing: bool = false

func _ready() -> void:
    _initialize_game()

func _initialize_game() -> void:
    # Grid
    grid = Grid.new()
    grid.grid_size = grid_size
    grid.num_colors = num_colors
    grid.init_grid()

    # ColorQueue
    color_queue = ColorQueue.new()
    color_queue.num_colors = num_colors

    # TurnManager
    turn_manager = TurnManager.new()
    turn_manager.mode = mode
    turn_manager.max_turns = max_turns

    # UI setup
    grid_view.setup(grid)
    color_queue_ui.setup(color_queue)
    hud.setup(mode, 1, goal, max_turns)

    # Signal connections
    grid_view.cell_touched.connect(_on_cell_touched)
    turn_manager.turn_changed.connect(hud.update_turns)

func _on_cell_touched(x: int, y: int) -> void:
    if _processing:
        return

    var cell = grid.get_cell(x, y)
    if cell == null or cell.color == -1:
        return

    var active_color = color_queue.get_active_color()

    # Same-color guard
    if cell.color == active_color:
        print("Same color — no action")
        return

    # === 유효한 액션 시작 ===
    _processing = true
    grid_view.lock_input()

    # 1. Turn
    turn_manager.use_turn()

    # 2. BFS Recolor
    var group = FloodFill.flood_fill(grid, x, y)
    for c in group:
        grid.set_cell_color(c.x, c.y, active_color)
    grid_view.refresh()

    # 3. ColorQueue advance
    color_queue.advance()
    color_queue_ui.refresh()

    # 4. Chain Combo
    var result = ChainCombo.execute(grid)
    _total_destroyed += result.total_destroyed

    # 5. HUD 갱신
    hud.update_destroyed(result.total_destroyed)
    if result.chain_count > 1:
        hud.show_chain(result.chain_count)
    grid_view.refresh()

    # 6. Clear/GameOver 판정
    _check_game_state()

    _processing = false
    grid_view.unlock_input()

func _check_game_state() -> void:
    if mode == "story":
        if _total_destroyed >= goal:
            _show_result("Stage Clear!")
            game_cleared.emit()
        elif turn_manager.turns_remaining <= 0:
            _show_result("Game Over")
            game_over.emit()
```

**테스트 시나리오 체크리스트**:
