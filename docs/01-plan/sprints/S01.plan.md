## T2: Cell Data Class (`cell.gd`)

### 설명

grid의 개별 셀을 표현하는 순수 데이터 클래스를 구현한다. 이 클래스는 Node를 상속하지 않는 `RefCounted` 기반 데이터 객체이다. 렌더링 로직 없이 데이터만 보유한다.

### Acceptance Criteria

- [ ] `Cell.new(x, y, color)` 로 인스턴스 생성 가능
- [ ] `color: int` — 색상 인덱스 (0 ~ num_colors-1), -1 = empty
- [ ] `x: int`, `y: int` — grid 좌표 (y=0이 main grid 최상단, y가 음수이면 buffer 영역)
- [ ] `active: bool` — `true`이면 main grid, `false`이면 buffer 또는 비활성
- [ ] `gimmick_type: int = -1` — 향후 기믹 확장용, -1 = none (이번 sprint에서는 미사용)
- [ ] `duplicate() -> Cell` — deep copy 메서드
- [ ] `equals(other: Cell) -> bool` — 좌표 + 색상 비교
- [ ] `to_string() -> String` — 디버그용 문자열 표현

### Dependencies

- T1 (git init 완료 후 파일 생성)

### Implementation Notes

```
# cell.gd 구조 스케치
class_name Cell
extends RefCounted

var x: int          # column index (0-based, left to right)
var y: int          # row index (0 = main grid top, negative = buffer)
var color: int      # color index, -1 = empty
var active: bool    # true = main grid area
var gimmick_type: int = -1  # reserved for future gimmick system

func _init(p_x: int = 0, p_y: int = 0, p_color: int = -1, p_active: bool = true) -> void:
    x = p_x
    y = p_y
    color = p_color
    active = p_active
```

- **좌표 시스템 규약**: `y` 값 기준으로:
  - `y >= 0` : main grid 영역 (y=0이 맨 위, y=grid_size-1이 맨 아래)
  - `y < 0`  : buffer 영역 (y=-1이 buffer 최하단, y=-grid_size이 buffer 최상단)
  - 이 규약은 S02 BFS에서 buffer 포함 탐색 시 중요
- `color` 값 범위: `0` ~ `num_colors - 1` (기본 num_colors = 5). `-1`은 empty 셀
- `gimmick_type`은 이번 milestone에서 사용하지 않지만, 데이터 구조에 포함하여 향후 확장 대비
- `RefCounted` 상속으로 메모리 자동 관리

### Output Files

- `scripts/core/cell.gd`

## T3: Grid Data Model (`grid.gd`)

### 설명

N x N main grid + N-row buffer를 관리하는 핵심 데이터 모델. Cell 객체들의 2D 배열을 보유하며, 셀 접근(CRUD), 초기화, 완성 라인 방지 로직을 포함한다. 이 클래스도 Node를 상속하지 않는 `RefCounted` 기반이다.

### Acceptance Criteria

- [ ] `@export var grid_size: int = 7` — N 값 (main grid = N x N, buffer = N rows x N cols)
- [ ] `@export var num_colors: int = 5` — 사용할 색상 수
- [ ] `init_grid()` 호출 시 (N x N) + (N x N) 총 2N x N 셀이 랜덤 색상으로 초기화
- [ ] `get_cell(x: int, y: int) -> Cell` — 유효 좌표면 Cell 반환, 아니면 null
- [ ] `set_cell_color(x: int, y: int, color: int)` — 셀 색상 변경
- [ ] `is_valid_coord(x: int, y: int) -> bool` — main + buffer 범위 체크
- [ ] `is_main_area(x: int, y: int) -> bool` — main grid 범위 체크
- [ ] `get_row(y: int) -> Array[Cell]` — 특정 row의 모든 셀
- [ ] `get_column(x: int) -> Array[Cell]` — 특정 column의 main area 셀만
- [ ] `ensure_no_completed_lines()` — init 후 완성된 row/column이 없도록 재배치
- [ ] 단위 테스트: `grid_size=7`로 init 후 14 x 7 = 98개 셀 존재 확인

### Dependencies

- T2 (`cell.gd` 필요)

### Implementation Notes

**내부 데이터 구조**:
```
# 2D Array: _cells[y][x]
# y range: -grid_size ~ grid_size-1
# x range: 0 ~ grid_size-1
# 내부적으로는 Dictionary 또는 offset array 사용

var _cells: Dictionary = {}  # key: Vector2i(x, y), value: Cell
```

- Dictionary 기반 `Vector2i(x, y) -> Cell` 매핑을 권장. 2D Array보다 음수 인덱스 처리가 간단함
- 대안: offset 방식 `_cells[y + grid_size][x]` 로 1D 배열 사용 가능하지만, 가독성 저하

**`ensure_no_completed_lines()` 알고리즘**:
1. main area의 모든 row (y=0 ~ grid_size-1) 검사
2. 모든 column (x=0 ~ grid_size-1) 검사 (main area만)
3. 완성된 라인 발견 시: 해당 라인의 랜덤 셀 1개를 다른 색으로 변경
4. 변경 후 다시 전체 검사 반복 (최대 100회, 무한루프 방지)
5. "완성된 라인" = 해당 라인의 모든 셀이 같은 색상

**라인 완성 판정 기준**:
- Row: `y` 값이 동일한 main area 셀 N개가 모두 같은 color
- Column: `x` 값이 동일한 main area 셀 N개가 모두 같은 color
- **Buffer 영역은 라인 완성 판정 대상이 아님**

**초기화 순서**:
1. `init_grid()` → 모든 셀에 `randi() % num_colors` 배정
2. `ensure_no_completed_lines()` 호출
3. 완료

**Edge Cases**:
- `grid_size < 3` 인 경우 경고 출력 (의미있는 게임 불가)
- `num_colors < 2` 인 경우 에러 (라인 완성 방지 불가능)
- `ensure_no_completed_lines()`에서 색상 변경 시, 현재 색상과 다른 랜덤 색 선택

### Output Files

- `scripts/core/grid.gd`

## T4: Grid Rendering Scene (`game.tscn`)

### 설명

`grid.gd`의 데이터를 화면에 시각적으로 렌더링하는 scene을 구성한다. 외부 에셋 없이 Godot 내장 노드(ColorRect, Control)만 사용한다. GridView라는 커스텀 Control 노드가 grid 데이터를 받아 셀들을 배치한다.

### Acceptance Criteria

- [ ] `game.tscn` 실행 시 7x7 main grid + 7-row buffer가 화면에 표시
- [ ] 각 셀이 색상에 따라 다른 `ColorRect` 색으로 표시
- [ ] 셀 간 1~2px gap 존재 (grid 구분선 역할)
- [ ] main grid와 buffer 영역 사이 시각적 구분선(또는 간격) 존재
- [ ] 화면 중앙에 grid가 위치
- [ ] grid_size 변경 시 (예: 5, 6, 7) 자동으로 셀 크기 조정
- [ ] 색상 팔레트: 최소 5색 구분 가능한 색상 배열 정의

### Dependencies

- T3 (`grid.gd` 데이터 모델 필요)

### Implementation Notes

**Scene 구조**:
```
game.tscn
├── GameManager (Node2D) — scripts/game_manager.gd (이 sprint에서는 최소 로직)
│   ├── GridView (Control) — scripts/ui/grid_view.gd
│   │   └── (동적으로 ColorRect children 생성)
│   └── Camera2D (선택사항)
```

**GridView 구현 방식**:
```
# grid_view.gd
class_name GridView
extends Control

@export var cell_size: int = 48      # 셀 한 변 크기 (px)
@export var cell_gap: int = 2        # 셀 간 간격 (px)
@export var buffer_gap: int = 8      # main/buffer 구분 간격 (px)

var _cell_rects: Dictionary = {}     # Vector2i -> ColorRect
```

**색상 팔레트 정의** (5색 + 확장 가능):
```
const COLOR_PALETTE: Array[Color] = [
    Color(0.91, 0.30, 0.24),   # 빨강 (Alizarin)
    Color(0.20, 0.60, 0.86),   # 파랑 (Peter River)
    Color(0.18, 0.80, 0.44),   # 초록 (Emerald)
    Color(0.95, 0.77, 0.06),   # 노랑 (Sun Flower)
    Color(0.56, 0.27, 0.68),   # 보라 (Amethyst)
    Color(1.00, 0.60, 0.20),   # 주황 (확장용)
    Color(0.20, 0.80, 0.80),   # 청록 (확장용)
]
```

**셀 위치 계산**:
```
func _cell_position(x: int, y: int) -> Vector2:
    var px = x * (cell_size + cell_gap)
    var py: float
    if y < 0:
        # buffer area: y=-1 이 buffer 최하단
        py = (y + grid_size) * (cell_size + cell_gap) - buffer_gap
    else:
        # main area
        py = (y + grid_size) * (cell_size + cell_gap)
    return Vector2(px, py)
```

**렌더링 업데이트 흐름**:
1. `setup(grid: Grid)` — Grid 데이터 참조 저장, ColorRect children 일괄 생성
2. `refresh()` — 모든 셀의 색상을 Grid 데이터 기준으로 갱신
3. `refresh_cell(x: int, y: int)` — 단일 셀 색상 갱신 (최적화용)

**화면 중앙 정렬**:
- GridView의 `position`을 viewport 크기 기준으로 계산:
  ```
  var total_width = grid_size * (cell_size + cell_gap) - cell_gap
  var total_height = (grid_size * 2) * (cell_size + cell_gap) - cell_gap + buffer_gap
  position = (get_viewport_rect().size - Vector2(total_width, total_height)) / 2
  ```

**주의사항**:
- ColorRect는 child node로 추가. `_cell_rects` Dictionary로 참조 관리
- 매 프레임 전체 refresh하지 말 것. 데이터 변경 시에만 `refresh()` 호출
- Godot 4.x에서는 `ColorRect.color` 속성으로 직접 색상 변경

### Output Files

- `scenes/game/game.tscn`
- `scripts/ui/grid_view.gd`
- `scripts/game_manager.gd` (최소 scaffold — GridView 연결만)

## T5: Cell Touch Input

### 설명

사용자가 grid의 셀을 터치(모바일) 또는 클릭(데스크톱)하면 해당 셀의 grid 좌표 `(x, y)`를 signal로 발신하는 입력 처리 시스템을 구현한다.

### Acceptance Criteria

- [ ] main grid 셀 클릭 시 `cell_touched(x: int, y: int)` signal 발신
- [ ] buffer 영역 클릭 시 signal 발신하지 않음 (main area만 터치 가능)
- [ ] grid 외부 클릭 시 무시
- [ ] 디버그 출력: 클릭된 좌표를 `print()` 로 확인 가능
- [ ] 마우스 클릭 + 터치스크린 모두 지원
- [ ] 연속 빠른 클릭 시 중복 처리 방지 (input lock flag)

### Dependencies

- T4 (GridView 렌더링 완료 필요 — 셀 위치 계산 로직 재사용)

### Implementation Notes

**좌표 변환 로직**:
```
# grid_view.gd 내부
signal cell_touched(x: int, y: int)

var _input_locked: bool = false

func _input(event: InputEvent) -> void:
    if _input_locked:
        return
    if event is InputEventMouseButton and event.pressed and event.button_index == MOUSE_BUTTON_LEFT:
        _handle_touch(event.position)
    elif event is InputEventScreenTouch and event.pressed:
        _handle_touch(event.position)

func _handle_touch(screen_pos: Vector2) -> void:
    var local_pos = screen_pos - global_position
    var gx = int(local_pos.x / (cell_size + cell_gap))
    var gy_raw = int(local_pos.y / (cell_size + cell_gap))
    var gy = gy_raw - grid_size  # offset 보정 (buffer 고려)

    # buffer_gap 보정: main area 시작점 이후인지 체크
    # main area only
    if gx < 0 or gx >= grid_size:
        return
    if gy < 0 or gy >= grid_size:
        return

    cell_touched.emit(gx, gy)
```

**Input Lock 메커니즘**:
- `_input_locked = true` → 애니메이션/처리 중 입력 차단
- `unlock_input()` → 처리 완료 후 해제
- S02에서 recolor → destroy → gravity 파이프라인 동안 lock 사용

**좌표 변환 주의사항**:
- `buffer_gap` 때문에 단순 나눗셈으로는 main/buffer 경계에서 오차 발생 가능
- 정확한 방법: main area의 pixel 시작점을 계산하고, 그 기준으로 offset
  ```
  var main_area_top_y = grid_size * (cell_size + cell_gap) + buffer_gap
  var rel_y = local_pos.y - main_area_top_y
  if rel_y < 0:
      return  # buffer area or gap — 무시
  var gy = int(rel_y / (cell_size + cell_gap))
  ```
- 셀과 셀 사이 gap 영역 클릭 시에도 가장 가까운 셀로 처리하거나 무시 — **무시 방식 권장** (gap 포함 계산이 더 안전)

**디버그 확인**:
- `cell_touched` signal을 GameManager에서 connect하여 `print("Touched: ", x, ", ", y)` 출력
- 향후 S02에서 이 signal에 BFS recolor 로직 연결

### Output Files

- `scripts/ui/grid_view.gd` (기존 파일에 입력 처리 추가)

## T6: Buffer Area Rendering

### 설명

main grid 위에 buffer 영역(N rows)을 시각적으로 구분하여 렌더링한다. Buffer 셀은 main 셀보다 투명하거나 크기가 작아 시각적으로 "대기 영역"임을 표현한다.

### Acceptance Criteria

- [ ] buffer 영역 (y = -grid_size ~ -1) 셀이 main grid 위에 표시
- [ ] buffer 셀은 main 셀 대비 약 50% 투명도 (alpha = 0.5)
- [ ] buffer 셀은 main 셀과 동일 크기이되, 색상 alpha 차이로 구분
- [ ] main grid와 buffer 사이 `buffer_gap` (8px) 간격 존재
- [ ] buffer 영역 최상단 위로 fade-out 효과 (선택사항, P2)
- [ ] grid_size 변경 시 buffer rows 수도 자동 조정

### Dependencies

- T4 (GridView 기본 렌더링 완료 필요)

### Implementation Notes

**Buffer 셀 시각 처리**:
```
func _create_cell_rect(x: int, y: int, color_index: int) -> ColorRect:
    var rect = ColorRect.new()
    rect.size = Vector2(cell_size, cell_size)
    rect.position = _cell_position(x, y)

    var base_color = COLOR_PALETTE[color_index]
    if y < 0:  # buffer area
        rect.color = Color(base_color.r, base_color.g, base_color.b, 0.5)
    else:       # main area
        rect.color = base_color

    return rect
```

**선택적 fade-out 효과** (P2):
- buffer 최상단 rows(y = -grid_size, -grid_size+1)는 alpha를 더 낮춤 (0.2 ~ 0.3)
- 점진적 alpha: `alpha = 0.2 + 0.3 * (float(y + grid_size) / grid_size)`

**구분선 옵션**:
- `buffer_gap` 간격으로 충분하지만, 시각적 명확성을 위해 1px Line2D 또는 ColorRect 구분선 추가 가능
- 구분선 색상: `Color(1, 1, 1, 0.3)` (흰색 반투명)

**Buffer 셀과 main 셀의 데이터 동기화**:
- `refresh()` 호출 시 buffer 셀도 함께 갱신
- S02 gravity에서 buffer → main으로 셀이 이동하면 alpha가 0.5 → 1.0으로 변경됨

### Output Files

- `scripts/ui/grid_view.gd` (기존 파일에 buffer 렌더링 로직 추가)
