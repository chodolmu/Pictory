## Task 1: DialogueManager

### 설명
스토리 다이얼로그를 진행시키는 핵심 매니저 스크립트. JSON 스토리 데이터를 로드하고, 한 줄씩 텍스트를 표시하며, 캐릭터 표시(좌/우), 타이핑 애니메이션, 자동 진행/탭 진행을 처리한다.

### 파일 경로
- `scripts/story/dialogue_manager.gd`

### 상세 구현

#### 핵심 API
```gdscript
# 다이얼로그 시퀀스 시작
func start_dialogue(story_data: Array[Dictionary]) -> void

# 다음 대사로 진행 (탭/클릭 시 호출)
func advance() -> void

# 전체 스킵
func skip_all() -> void

# 현재 진행 중인지
func is_active() -> bool
```

#### 시그널
```gdscript
signal dialogue_started
signal dialogue_line_shown(line_data: Dictionary)
signal dialogue_finished
signal event_triggered(event_name: String, event_args: Dictionary)
```

#### 타이핑 애니메이션
- `RichTextLabel`의 `visible_characters` 속성을 Tween으로 증가
- 타이핑 속도: `@export var typing_speed: float = 30.0` (초당 글자 수)
- 타이핑 중 탭하면 즉시 전체 표시 (skip typing)
- 전체 표시 상태에서 탭하면 다음 대사로 진행

#### 캐릭터 표시 로직
- `position` 필드에 따라 좌/우 캐릭터 영역 활성화
- 현재 말하는 캐릭터: 밝게 (modulate = Color.WHITE)
- 말하지 않는 캐릭터: 어둡게 (modulate = Color(0.5, 0.5, 0.5, 1.0))
- 캐릭터 등장/퇴장: Tween 슬라이드 (화면 밖에서 안으로)

#### 표정(expression) 시스템
- AI-only 원칙: 캐릭터는 도형(ColorRect + 원/사각형 조합)으로 표현
- expression 값에 따라 눈/입 도형 변경:
  - `"normal"`: 기본 표정
  - `"happy"`: 눈 = 반원 위쪽, 입 = 위로 곡선
  - `"sad"`: 눈 = 반원 아래쪽, 입 = 아래로 곡선
  - `"angry"`: 눈 = 사선, 입 = 일자
  - `"surprised"`: 눈 = 큰 원, 입 = 작은 원
- 표정 전환은 Tween 보간으로 부드럽게

#### 이벤트 트리거
- 대사 라인에 `"event"` 필드가 있으면 `event_triggered` 시그널 발행
- 이벤트 예시: `"screen_shake"`, `"flash"`, `"bgm_change"`, `"imagen_join"`
- 이벤트 처리는 DialogueManager 외부에서 시그널 연결로 수행

### 수락 기준
- [ ] JSON 스토리 데이터를 로드하여 순서대로 대사를 표시한다
- [ ] 타이핑 애니메이션이 동작하며, 탭 시 즉시 표시 / 다음 진행이 된다
- [ ] 좌/우 캐릭터가 올바른 위치에 표시되고, 말하지 않는 쪽은 어둡게 처리된다
- [ ] expression에 따라 캐릭터 표정이 변한다
- [ ] 마지막 대사 이후 `dialogue_finished` 시그널이 발행된다
- [ ] event 필드가 있는 라인에서 `event_triggered` 시그널이 발행된다

### 의존성
- 없음 (독립적으로 개발 가능)

### 구현 노트
- `dialogue_manager.gd`는 씬 트리에 직접 배치하지 않고, `dialogue.tscn`의 스크립트에서 인스턴스화하거나, Autoload로 등록하여 사용
- 스토리 데이터는 Array[Dictionary] 형태로 전달받으므로, JSON 파싱은 별도 로더에서 처리

## Task 2: 다이얼로그 UI 씬

### 설명
스토리 다이얼로그를 표시하는 전체 UI 씬. 캐릭터 표시 영역(좌/우), 텍스트 박스, 이름표, 스킵 버튼, 나가기 버튼으로 구성.

### 파일 경로
- `scenes/story/dialogue.tscn`
- `scripts/story/dialogue_ui.gd` (씬 컨트롤러)

### 씬 트리 구조
```
DialogueUI (CanvasLayer)
├── Background (ColorRect) — 반투명 배경
├── CharacterLeft (Control)
│   ├── Body (ColorRect) — 캐릭터 몸체
│   ├── Face (Control) — 표정 도형 컨테이너
│   │   ├── EyeLeft, EyeRight, Mouth
│   └── NameBadge (PanelContainer + Label) — 이름 뱃지
├── CharacterRight (Control)
│   ├── Body (ColorRect)
│   ├── Face (Control)
│   │   ├── EyeLeft, EyeRight, Mouth
│   └── NameBadge (PanelContainer + Label)
├── TextBox (PanelContainer)
│   ├── NameLabel (Label) — 현재 화자 이름
│   ├── DialogueText (RichTextLabel) — 대사 텍스트
│   └── NextIndicator (TextureRect/Label) — "▼" 다음 표시 (깜빡임 Tween)
├── SkipButton (Button) — "[스토리 스킵]"
└── ExitButton (Button) — "[나가기]"
```

### 상세 구현

#### 캐릭터 표현 (AI-only)
- 캐릭터 몸체: `ColorRect` (80×120px 정도)
  - 후냐(Hunya): 흰색 배경 + 갈색 귀 도형 (상단 삼각형 2개)
  - 이마젠: 속성별 색상 (불=빨강, 물=파랑, 풀=초록 등) + 특징 도형
- 표정: `Face` 노드 아래 눈/입 도형을 expression에 따라 변경
- 캐릭터 크기: 화면 높이의 약 30%

#### 텍스트 박스
- 화면 하단 80% 너비, 높이 약 120px
- 배경: 반투명 어두운 PanelContainer (StyleBoxFlat)
- 이름표: 텍스트 박스 좌상단 위에 걸치는 형태, 화자 색상 배경
- 대사 텍스트: `RichTextLabel`, BBCode 지원 (색상 강조 등)
- "▼" 표시: 타이핑 완료 후 우하단에서 깜빡임

#### 스킵/나가기 버튼
- `[스토리 스킵]`: 우상단, 누르면 확인 팝업 → 전체 다이얼로그 스킵 → 게임으로 진행
- `[나가기]`: 좌상단, 누르면 확인 팝업 → 스테이지 선택 화면으로 복귀
- 스킵 확인 팝업: "스토리를 건너뛰시겠습니까?" + `[확인]` + `[취소]`

#### 입력 처리
- 텍스트 박스 영역 또는 빈 화면 탭 → `DialogueManager.advance()` 호출
- 스킵/나가기 버튼 위 탭은 대사 진행으로 처리하지 않음

### 수락 기준
- [ ] 캐릭터가 좌/우 위치에 도형으로 렌더링된다
- [ ] 이름표에 현재 화자 이름이 표시된다
- [ ] 대사가 타이핑 애니메이션과 함께 표시된다
- [ ] 타이핑 완료 후 "▼" 표시가 깜빡인다
- [ ] `[스토리 스킵]` → 확인 → 전체 스킵 동작
- [ ] `[나가기]` → 확인 → 스테이지 선택으로 복귀
- [ ] 캐릭터 등장 시 슬라이드 Tween이 동작한다

### 의존성
- Task 1 (DialogueManager)

### 구현 노트
- CanvasLayer를 사용하여 게임 씬 위에 오버레이 가능하도록 설계
- 캐릭터 도형은 별도 씬(`character_display.tscn`)으로 분리하면 재사용성 향상
- 캐릭터별 외형 정의는 Dictionary 또는 Resource 파일로 관리

## Task 3: 스토리 연출 화면

### 설명
UI 플로우에서 "스토리 연출" 단계를 담당하는 씬. 다이얼로그 UI를 포함하며, 스토리 JSON을 로드하여 다이얼로그를 진행하고, 완료 시 다음 단계(게임 또는 결과)로 전환한다.

### 파일 경로
- `scenes/story/story_screen.tscn`
- `scripts/story/story_screen.gd`

### 상세 구현

#### 역할
- SceneManager로부터 전환 시 스토리 데이터 경로와 유형(pre/post)을 받음
- JSON을 로드하여 DialogueManager에 전달
- 다이얼로그 종료 시그널 수신 → 다음 씬으로 전환 요청

#### 초기화 파라미터
```gdscript
var story_params: Dictionary = {
    "chapter": 1,
    "stage": 1,
    "type": "pre",  # "pre" | "post"
    "next_scene": "res://scenes/game/game.tscn",  # 다이얼로그 후 이동할 씬
    "next_params": {}  # 다음 씬에 전달할 파라미터
}
```

#### 스토리 파일 경로 규칙
```
resources/story/chapter_{NN}/stage_{NN}_{type}.json
예: resources/story/chapter_01/stage_01_pre.json
    resources/story/chapter_01/stage_01_post.json
```

#### 이벤트 처리
- `event_triggered` 시그널에서 연출 이벤트 처리:
  - `"screen_shake"`: 화면 흔들림 (Camera2D offset Tween)
  - `"flash_white"`: 백색 플래시 (ColorRect alpha 0→1→0)
  - `"flash_black"`: 흑색 플래시
  - `"bgm_change"`: BGM 전환 (AudioStreamPlayer, S14에서 실제 구현)
  - `"wait"`: N초 대기 후 자동 진행
  - `"imagen_join"`: 이마젠 합류 이벤트 (S09에서 연결)

### 수락 기준
- [ ] story_params를 받아 올바른 JSON 파일을 로드한다
- [ ] 다이얼로그 완료 후 next_scene으로 전환된다
- [ ] pre 다이얼로그 → 게임 → post 다이얼로그 흐름이 동작한다
- [ ] 스킵 시에도 다음 씬으로 정상 전환된다
- [ ] screen_shake, flash 등 연출 이벤트가 동작한다

### 의존성
- Task 1 (DialogueManager)
- Task 2 (다이얼로그 UI 씬)
- S05 SceneManager (`scripts/ui/scene_manager.gd`)

### 구현 노트

- 스토리 파일이 없는 스테이지는 빈 배열 `[]`로 처리 → 즉시 다음 씬으로 전환

## Task 4: 스토리 데이터 JSON 스키마

### 설명
스토리 대사 데이터의 JSON 스키마를 정의한다. 모든 챕터/스테이지의 스토리 스크립트가 이 스키마를 따른다.

### 파일 경로
- `resources/story/schema.json` (스키마 정의 참조용)

### JSON 스키마

#### 파일 구조
```json
{
    "chapter": 1,
    "stage": 1,
    "type": "pre",
    "lines": [
        {
            "speaker": "hunya",
            "name_display": "후냐",
            "text": "여기가 바로 그 장소야!",
            "position": "left",
            "expression": "happy",
            "event": null
        },
        {
            "speaker": "imagen_fire",
            "name_display": "불꽃이",
            "text": "위험한 기운이 느껴져...",
            "position": "right",
            "expression": "sad",
            "event": {
                "type": "screen_shake",
                "args": { "intensity": 5, "duration": 0.5 }
            }
        },
        {
            "speaker": "narrator",
            "name_display": "",
            "text": "어둠의 기운이 퍼즐 세계를 감싸고 있었다...",
            "position": "center",
            "expression": null,
            "event": {
                "type": "flash_black",
                "args": { "duration": 1.0 }
            }
        }
    ]
}
```

#### 필드 정의

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `chapter` | int | O | 챕터 번호 |
| `stage` | int | O | 스테이지 번호 |
| `type` | string | O | `"pre"` 또는 `"post"` |
| `lines` | Array | O | 대사 라인 배열 |
| `lines[].speaker` | string | O | 화자 식별자 (예: `"hunya"`, `"imagen_fire"`, `"narrator"`) |
| `lines[].name_display` | string | O | UI에 표시할 이름 (내레이터는 빈 문자열) |
| `lines[].text` | string | O | 대사 텍스트. BBCode 지원 (`[color=red]강조[/color]`) |
| `lines[].position` | string | O | `"left"`, `"right"`, `"center"` (내레이터용) |
| `lines[].expression` | string | X | 표정: `"normal"`, `"happy"`, `"sad"`, `"angry"`, `"surprised"`. null이면 이전 표정 유지 |
| `lines[].event` | object\|null | X | 연출 이벤트. null이면 이벤트 없음 |
| `lines[].event.type` | string | O* | 이벤트 타입 |
| `lines[].event.args` | object | X | 이벤트 인자 |

#### speaker 목록 (챕터 1 기준)
- `"hunya"` — 후냐 (주인공, 좌측 기본)
- `"imagen_fire"` — 불 속성 이마젠 (예시)
- `"imagen_water"` — 물 속성 이마젠 (예시)
- `"narrator"` — 내레이터 (캐릭터 표시 없음, 텍스트만)
- `"unknown"` — 미지의 존재 (실루엣 표시)

#### 이벤트 타입 목록

| 이벤트 | args | 설명 |
|--------|------|------|
| `screen_shake` | `{ intensity: float, duration: float }` | 화면 흔들림 |
| `flash_white` | `{ duration: float }` | 백색 플래시 |
| `flash_black` | `{ duration: float }` | 흑색 플래시 |
| `bgm_change` | `{ track: string }` | BGM 전환 |
| `wait` | `{ seconds: float }` | 자동 대기 |
| `imagen_join` | `{ imagen_id: string }` | 이마젠 합류 (S09) |
| `chapter_clear` | `{}` | 챕터 클리어 연출 |

### 수락 기준
- [ ] 스키마 정의 문서가 작성되어 있다
- [ ] JSON 파일이 스키마에 맞게 파싱된다
- [ ] 필수 필드 누락 시 에러 로그를 출력한다
- [ ] 알 수 없는 speaker/event 시 기본값으로 폴백한다

### 의존성
- 없음

### 구현 노트
- JSON 파싱 시 `JSON.parse_string()` 사용
- 유효성 검증은 `_validate_line(line: Dictionary) -> bool` 함수로 구현
- 빈 lines 배열(`[]`)도 유효 — 즉시 종료 처리

## Task 5: 챕터 타이틀 카드

### 설명
새 챕터에 처음 진입할 때 표시되는 타이틀 카드 연출. 챕터 번호와 제목을 화면 중앙에 표시하고, Tween 애니메이션으로 등장/퇴장한다.

### 파일 경로
- `scenes/story/chapter_title.tscn`
- `scripts/story/chapter_title.gd`

### 씬 트리 구조
```
ChapterTitle (CanvasLayer)
├── Background (ColorRect) — 전체 화면 단색 배경 (검정)
├── ChapterNumber (Label) — "Chapter 1" (영문 또는 "제1장")
├── ChapterName (Label) — 챕터 제목 (예: "퍼즐 세계에 오신 것을 환영합니다")
└── Decoration (Control) — 장식 라인/도형 (AI-only)
```

### 상세 구현

#### 애니메이션 시퀀스 (총 3초)
1. **0.0s**: 검은 화면
2. **0.0~0.5s**: 챕터 번호 페이드인 (alpha 0→1)
3. **0.5~1.0s**: 챕터 제목 아래에서 슬라이드 업 + 페이드인
4. **1.0~1.5s**: 장식 라인 좌우에서 펼쳐짐
5. **1.5~2.5s**: 정지 (읽는 시간)
6. **2.5~3.0s**: 전체 페이드아웃

#### API
```gdscript
signal title_finished

func show_title(chapter_number: int, chapter_name: String) -> void
```

#### 탭으로 스킵


## Task 7: 스토리→게임 플로우 연결

### 설명
스토리 다이얼로그와 게임 플레이를 매끄럽게 연결하는 플로우 로직을 구현한다. 스테이지 선택 → (챕터 타이틀) → pre 다이얼로그 → 게임 → 클리어/실패 → (post 다이얼로그) → 결과.

### 파일 경로
- `scripts/story/story_flow_controller.gd` (새 파일 또는 SceneManager 확장)
- `scripts/ui/scene_manager.gd` (수정)
- `scripts/modes/story_mode.gd` (수정)

### 상세 구현

#### 전체 플로우 상태 머신
```
CHAPTER_TITLE → PRE_DIALOGUE → GAME → POST_DIALOGUE → RESULT
                                  └─ (실패) → RESULT (post 다이얼로그 스킵)
```

#### StoryFlowController
```gdscript
enum FlowState {
    IDLE,
    CHAPTER_TITLE,
    PRE_DIALOGUE,
    GAME,
    POST_DIALOGUE,
    RESULT
}

var current_state: FlowState = FlowState.IDLE
var chapter: int
var stage: int
var is_first_visit: bool  # 챕터 첫 방문 여부

func start_stage(p_chapter: int, p_stage: int) -> void:
    chapter = p_chapter
    stage = p_stage
    is_first_visit = _check_first_visit()
    
    if is_first_visit and stage == 1:
        _show_chapter_title()
    else:
        _show_pre_dialogue()

func _on_chapter_title_finished() -> void:
    _show_pre_dialogue()

func _on_pre_dialogue_finished() -> void:
    _start_game()

func _on_game_finished(result: Dictionary) -> void:
    if result.is_clear:
        _show_post_dialogue()
    else:
        _show_result(result)

func _on_post_dialogue_finished() -> void:
    _show_result(_last_game_result)
```

#### SceneManager 수정사항
- `change_scene(path, params)` 지원 추가 (params를 다음 씬의 `_ready`에서 접근 가능하도록)
- 씬 전환 시 Tween 페이드 트랜지션 유지

#### GameManager/StoryMode 수정사항
- 게임 종료 시 `game_finished` 시그널에 결과 Dictionary 포함:
  ```gdscript
  signal game_finished(result: Dictionary)
  # result = { is_clear: bool, stars: int, score: int, destroyed: int, ... }
  ```
- `StoryFlowController`가 이 시그널을 수신

#### pre 다이얼로그 없는 스테이지 처리
- JSON 파일이 없거나 lines가 빈 배열이면 즉시 다음 단계로
- `FileAccess.file_exists()` 체크

### 수락 기준
- [ ] 스테이지 선택 → 챕터 타이틀(첫 방문) → pre 대화 → 게임 → post 대화 → 결과 흐름이 동작한다
- [ ] 챕터 첫 방문이 아니면 타이틀을 건너뛴다
- [ ] 게임 실패 시 post 다이얼로그 없이 바로 결과 팝업으로 간다
- [ ] 스토리 스킵 시에도 게임으로 정상 진행된다
- [ ] 나가기 시 스테이지 선택 화면으로 복귀한다
- [ ] 결과 팝업에서 "다음 스테이지" 선택 시 다음 스테이지의 pre 대화부터 시작한다
