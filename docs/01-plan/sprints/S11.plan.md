# S11: 수익화 — 행동력 + 상점 + 광고 (Week 11)

> **Milestone**: M5 — Collection & Economy
> **스프린트 목표**: 행동력 시스템, 상점, 광고/구매 팝업이 동작하여 수익화 기반이 완성된다
> **선행 조건**: S10 완료 (컬렉션/업적 시스템, CurrencyManager, SaveManager 모두 동작)

---

## 설계 개요

### 수익화 시스템 아키텍처

```
StaminaManager (Autoload)
├── 현재 행동력 (current_stamina: int)
├── 최대 행동력 (max_stamina: int = 30)
├── 자동 회복 타이머 (recovery_interval: float = 300.0초 = 5분)
├── 마지막 회복 시각 (last_recovery_time: int)
├── 오프라인 회복 계산
└── SaveManager 연동

AdManager (Autoload)
├── 광고 인터페이스 정의 (추상)
├── 개발용 스텁 (즉시 보상)
├── 광고 시청 가능 여부 체크
├── 일일 광고 시청 제한
└── 보상 콜백

ShopManager
├── 상품 데이터 로드
├── 구매 처리 (재화/행동력)
├── IAP 인터페이스 스텁
└── 구매 이력 기록
```

### UI 플로우 연동

```
메인 메뉴
├── 행동력 표시 (상단 바): ⚡ 25/30 + 다음 회복까지 타이머
├── [상점] 버튼 → 상점 화면
└── [스토리] / [무한] 진입 시 행동력 체크

스테이지 선택 팝업 (기존 S05)
├── 행동력 표시 추가
├── 행동력 충분 → [시작] 활성
├── 행동력 부족 → [시작] 비활성 → 행동력 부족 팝업
└── [광고/구매 포인트 추가] 버튼 → 광고/구매 팝업

무한 모드 진입 팝업
├── 행동력 표시 추가
├── 행동력 충분 → [시작] 활성
└── 행동력 부족 → 행동력 부족 팝업

행동력 부족 팝업
├── "행동력이 부족합니다"
├── [광고 시청] → AdManager → 행동력 +5
├── [구매] → 상점 화면으로 이동
└── [닫기]

광고/구매 전용 팝업
├── [광고 시청으로 보상 받기] → AdManager
├── [구매하기] → 상점/IAP
└── [팝업 닫기]
```

### 경제 밸런스 설계

```
행동력:
  - 최대: 30
  - 스토리 1판 소모: 1
  - 무한 1판 소모: 2
  - 자동 회복: 5분당 1 (최대까지 150분 = 2.5시간)
  - 광고 시청 보상: +5
  - 상점 구매: 10개 / 30개(풀충전) 패키지

재화 획득:
  - 스토리 클리어: 기본 50 + 별 보너스 (1성 +0, 2성 +25, 3성 +50) = 50~100
  - 무한 모드: score / 10 (1000점 → 100 재화)
  - 업적 보상: 100~500
  - 세트 완성 보상: 200~1000

상점 가격:
  - 행동력 10개: 100 재화
  - 행동력 30개(풀충전): 250 재화
  - 재화 500개: ₩1,100 (IAP 스텁)
  - 재화 1200개: ₩2,200 (IAP 스텁)
  - 재화 3000개: ₩4,400 (IAP 스텁)

챕터 해금 (재화 소모):
  - 챕터 2: 500 재화 (이전 챕터 올클리어 필요)
  - 챕터 3: 800 재화
  - 챕터 4: 1,200 재화
  - 챕터 5~10: 점진 증가 (S05 Task 11 참조)
  - 해금된 챕터 내 스테이지는 순차 진행, 스테이지별 추가 비용 없음
```

## Task 1: 행동력 시스템 (StaminaManager)

### 설명

게임의 핵심 게이팅 메커니즘인 행동력(스태미나) 시스템. 스테이지 시작 시 소모되고, 시간 경과에 따라 자동 회복되며, 광고 시청이나 재화 구매로 충전할 수 있다.

### 수용 기준 (Acceptance Criteria)

- [ ] Autoload `StaminaManager`로 등록
- [ ] `current_stamina: int` — 현재 행동력 (0 ~ max_stamina)
- [ ] `max_stamina: int = 30` — 최대 행동력
- [ ] `consume(amount: int) -> bool` — 행동력 소모. 부족 시 false 반환, 성공 시 true + 차감
- [ ] `add(amount: int)` — 행동력 추가 (max_stamina 초과 불가, 광고/구매 보상용)
- [ ] `can_play(mode: String) -> bool` — 모드별 필요 행동력 체크 (story=1, infinity=2)
- [ ] `get_time_to_next_recovery() -> int` — 다음 회복까지 남은 초
- [ ] 시간 자동 회복: `recovery_interval = 300.0`초 (5분)마다 1 회복
- [ ] 오프라인 회복: 앱 재시작 시 `last_recovery_time`과 현재 시각 비교 → 경과 시간만큼 회복
- [ ] `signal stamina_changed(current, max)` — 행동력 변경 시 시그널 발행
- [ ] `signal stamina_depleted()` — 행동력 0이 된 순간 시그널 발행
- [ ] SaveManager 연동: `current_stamina`, `last_recovery_time` 저장/로드
- [ ] 행동력이 max 미만일 때만 회복 타이머 동작

### 의존성

- `SaveManager` (S05) — 상태 저장/로드
- OS 시간 함수 — `Time.get_unix_time_from_system()`

### 주요 구현 노트

- 회복 타이머는 `Timer` 노드 또는 `_process(delta)` 내부 카운터로 구현
- 오프라인 회복 계산: `elapsed = current_time - last_recovery_time`, `recovered = int(elapsed / recovery_interval)`, `current_stamina = min(current_stamina + recovered, max_stamina)`
- 치트 방지는 1차 릴리즈에서는 고려하지 않음 (로컬 저장만)
- 테스트 편의를 위해 `@export var debug_unlimited: bool = false` 플래그 추가 — true 시 행동력 무제한
- 행동력 소모 시점: 스테이지 로딩 직전 (게임 화면 진입 시), 게임 오버/클리어와 무관하게 소모됨
- 챕터 기반 진행 모델: 챕터 해금은 재화로 처리 (S05 Task 11), 행동력은 개별 스테이지 플레이 시마다 소모. 해금된 챕터 내 스테이지는 순차 진행하되 각 스테이지 플레이에 행동력 1 필요

### 산출물 파일 경로

- `scripts/data/stamina_manager.gd`

## Task 2: 행동력 UI 표시

### 설명

메인 메뉴 상단 바와 스테이지/무한 모드 진입 팝업에 현재 행동력을 표시하는 UI 컴포넌트. 다음 회복까지의 타이머도 실시간으로 갱신된다.

### 수용 기준 (Acceptance Criteria)

- [ ] 메인 메뉴 상단 바에 행동력 표시: `⚡ 25/30` 형식
- [ ] 행동력이 max 미만일 때: 다음 회복까지 카운트다운 표시 (`다음 회복: 3:42`)
- [ ] 행동력이 max일 때: 카운트다운 숨김, "충전 완료" 표시
- [ ] 행동력 변경 시 즉시 UI 갱신 (StaminaManager.stamina_changed 시그널 연결)
- [ ] 행동력 낮을 때 시각적 경고: 5 이하일 때 텍스트 빨간색
- [ ] 스테이지 선택 팝업에 행동력 표시 추가 (기존 팝업 수정)
- [ ] 무한 모드 진입 팝업에 행동력 표시 추가 (기존 팝업 수정)
- [ ] 행동력 표시 탭 시 행동력 충전 안내 팝업 또는 상점으로 이동

### 의존성

- `StaminaManager` (Task 1) — 행동력 데이터
- 메인 메뉴 씬 (S05) — 상단 바 영역에 UI 추가
- 스테이지 선택 팝업 (S05) — 기존 팝업에 행동력 표시 추가
- 무한 모드 진입 팝업 (S05) — 기존 팝업에 행동력 표시 추가

### 주요 구현 노트

- 행동력 바 컴포넌트를 재사용 가능한 씬(`stamina_bar.tscn`)으로 만들어 여러 화면에 인스턴스화
- 구성: HBoxContainer > [아이콘 Label "⚡"] [현재/최대 Label] [타이머 Label]
- 타이머 갱신은 `_process(delta)`에서 매 프레임이 아닌 1초 간격으로 업데이트 (성능)
- 메인 메뉴 상단 바에는 재화 표시(기존)와 행동력 표시를 나란히 배치
- 색상: 정상(흰색), 경고(빨간색, 5 이하), 충전 완료(초록색)

### 산출물 파일 경로

- `scenes/ui/stamina_bar.tscn`
- `scripts/ui/stamina_bar.gd`
- 기존 수정: `scenes/ui/main_menu.tscn`, `scenes/ui/stage_select_popup.tscn`, `scenes/ui/infinity_popup.tscn`

## Task 3: 행동력 부족 팝업

### 설명

행동력이 부족한 상태에서 스테이지를 시작하려 할 때 표시되는 팝업. 광고 시청, 상점 이동, 닫기 옵션을 제공한다.

### 수용 기준 (Acceptance Criteria)

- [ ] 팝업 타이틀: "행동력이 부족합니다"
- [ ] 현재 행동력 / 필요 행동력 표시: "현재: 0 / 필요: 1"
- [ ] 다음 자동 회복 시간 표시: "다음 회복까지: 3분 42초"
- [ ] `[광고 시청]` 버튼 → AdManager.show_rewarded_ad() → 성공 시 행동력 +5 → 팝업 닫기
- [ ] `[구매]` 버튼 → 상점 화면으로 씬 전환 (또는 상점 팝업 표시)
- [ ] `[닫기]` 버튼 → 팝업 닫기 (아무 동작 없이 복귀)
- [ ] 광고 시청 후 행동력이 충분해지면 자동으로 게임 시작 진행 (선택적)
- [ ] 광고 일일 제한 도달 시 `[광고 시청]` 버튼 비활성 + "오늘 광고 시청 횟수를 모두 사용했습니다"

### 의존성

- `StaminaManager` (Task 1) — 행동력 상태
- `AdManager` (Task 6) — 광고 시청 인터페이스
- 상점 화면 (Task 4) — 구매 버튼 연결

### 주요 구현 노트

- 팝업은 모달 (배경 딤 처리)
- 행동력 부족 체크 시점: 스테이지/무한 모드 `[시작]` 버튼 클릭 → `StaminaManager.can_play()` false → 팝업 표시
- 기존 스테이지 선택 팝업과 무한 모드 팝업의 `[시작]` 버튼 핸들러에 행동력 체크 로직 삽입
- 광고 보상 콜백: `AdManager.signal ad_reward_granted` → `StaminaManager.add(5)` → UI 갱신 → 팝업 닫기

### 산출물 파일 경로

- `scenes/ui/stamina_depleted_popup.tscn`
- `scripts/ui/stamina_depleted_popup.gd`

## Task 4: 상점 화면

### 설명

재화와 행동력을 구매할 수 있는 상점 화면. 재화 구매(IAP)와 행동력 구매(재화 소모) 두 카테고리로 구성된다.

### 수용 기준 (Acceptance Criteria)

- [ ] 상점 화면 레이아웃:
  - 상단: "상점" 타이틀 + 현재 보유 재화 표시
  - 중앙: 상품 목록 (ScrollContainer)
  - 하단: `[뒤로]` 버튼
- [ ] 상품 카테고리 탭: `[재화]` `[행동력]` `[챕터]`
- [ ] **재화 탭** — IAP 상품 목록:
  - 재화 500개 — ₩1,100
  - 재화 1,200개 — ₩2,200 (가성비 배지)
  - 재화 3,000개 — ₩4,400 (최고 가성비 배지)
  - 각 상품: 아이콘 + 수량 + 가격 + `[구매]` 버튼
- [ ] **행동력 탭** — 재화로 구매:
  - 행동력 10개 — 100 재화
  - 행동력 30개 (풀충전) — 250 재화
  - 각 상품: 아이콘 + 수량 + 재화 가격 + `[구매]` 버튼
- [ ] **챕터 탭** — 재화로 챕터 해금:
  - 해금 가능한 다음 챕터 표시 (이전 챕터 올클리어 필요)
  - 챕터별 해금 비용 표시 (S05 Task 11의 CHAPTER_COSTS 참조)
  - `[해금]` 버튼 → ChapterUnlock.try_unlock() 호출
  - 이미 해금된 챕터는 "해금됨" 표시 (비활성)
  - 선행 조건 미충족 시 "챕터 N 올클리어 필요" 안내
- [ ] 재화 구매(IAP): 1차에서는 스텁 — `[구매]` 클릭 → 확인 팝업 → 즉시 재화 지급 (실제 결제 없음)
- [ ] 행동력 구매: 재화 부족 시 `[구매]` 버튼 비활성 + "재화가 부족합니다" 툴팁
- [ ] 구매 성공 시: 간단한 성공 애니메이션 (재화/행동력 수치 증가 Tween)
- [ ] 구매 이력 기록: `ShopManager`에 구매 횟수/날짜 저장 (향후 분석용)
- [ ] `[뒤로]` → 이전 화면으로 복귀

### 의존성

- `CurrencyManager` (S05) — 재화 잔액 확인 및 차감
- `StaminaManager` (Task 1) — 행동력 추가
- 메인 메뉴 (S05) — `[상점]` 버튼에서 진입

### 주요 구현 노트

- 상품 데이터는 `resources/shop/products.json`에 정의:
  ```json
  {
    "currency_products": [
      {"id": "currency_500", "amount": 500, "price_display": "₩1,100", "price_value": 1100}
    ],
    "stamina_products": [
      {"id": "stamina_10", "amount": 10, "cost_currency": 100}
    ],
    "chapter_products": [
      {"id": "chapter_2", "chapter": 2, "cost_currency": 500, "prerequisite": "chapter_1_clear"},
      {"id": "chapter_3", "chapter": 3, "cost_currency": 800, "prerequisite": "chapter_2_clear"}
    ]
  }
  ```
- 챕터 탭은 `ChapterUnlock` (S05 Task 11)과 연동하여 해금 조건/비용을 표시
- 챕터 해금은 상점 외에도 스테이지 선택 화면에서도 가능 (S05 Task 11)
- 상품 카드는 재사용 가능한 씬 `shop_item_card.tscn`으로 구현
- IAP 스텁: `IAPManager` 인터페이스 정의 + `DevIAPManager` 구현 (즉시 성공 반환)
- 추후 실제 IAP 연동 시 `DevIAPManager`를 `GooglePlayIAPManager` / `AppStoreIAPManager`로 교체
- 가성비 배지: 상품 카드에 조건부 표시 (amount/price 비율이 높은 상품)

## Task 6: 광고 매니저 스텁 (AdManager)

### 설명
