# S09: 이마젠 시스템 (Week 9)

> **마일스톤**: M4 — Story & Companion
> **목표**: 동료 이마젠을 편성하고 인게임에서 스킬을 사용할 수 있다
> **선행 조건**: S03(GameManager/턴 관리), S04(모드 로직), S05(UI 플로우/스테이지 확인 팝업), S07(기믹 완성), S08(스토리 시스템)
> **원칙**: AI-only — 이마젠 비주얼은 Godot 내장 도형/색상/셰이더/Tween/파티클로 구현

---

## 시스템 개요

```
[스테이지 선택] → [편성 팝업: 1~2 이마젠 선택] → [게임 시작]
                                                       │
                                                  HUD에 스킬 버튼 표시
                                                       │
                                              탭 → 스킬 발동 (턴 미소모)
                                                       │
                                              쿨타임 N턴 동안 비활성
                                                       │
                                              쿨타임 해제 → 재사용 가능
```

## Task 1: Imagen 데이터 구조

### 설명
이마젠의 데이터 구조를 정의하는 Resource 클래스. 이름, 속성, 보유 스킬, 쿨타임, 해금 조건 등을 포함.

### 파일 경로
- `scripts/companion/imagen.gd`
- `scripts/companion/imagen_database.gd` (이마젠 목록 관리)

### 상세 구현

#### ImagenData (Resource)
```gdscript
class_name ImagenData
extends Resource

@export var id: String              # 고유 식별자 (예: "fire_01")
@export var display_name: String    # 표시 이름 (예: "불꽃이")
@export var attribute: String       # 속성: "fire", "water", "grass", "light", "dark"
@export var description: String     # 설명 텍스트
@export var skill_id: String        # 보유 스킬 ID (예: "K1")
@export var cooldown: int           # 스킬 쿨타임 (턴 수)
@export var unlock_condition: Dictionary  # 해금 조건
# 예: { "type": "story_clear", "chapter": 1, "stage": 3 }
# 예: { "type": "achievement", "achievement_id": "combo_10" }

# 속성별 색상 (AI-only 비주얼용)
func get_color() -> Color:
    match attribute:
        "fire": return Color(1.0, 0.3, 0.2)
        "water": return Color(0.2, 0.5, 1.0)
        "grass": return Color(0.3, 0.8, 0.3)
        "light": return Color(1.0, 0.95, 0.6)
        "dark": return Color(0.4, 0.2, 0.6)
        _: return Color.WHITE
```

#### ImagenDatabase (Autoload)
```gdscript
class_name ImagenDatabase
extends Node

# 전체 이마젠 목록 (JSON에서 로드)
var all_imagenes: Dictionary = {}  # id -> ImagenData

# 해금된 이마젠 목록 (SaveManager 연동)
var unlocked_ids: Array[String] = []

func load_database() -> void
func get_imagen(id: String) -> ImagenData
func is_unlocked(id: String) -> bool
func unlock(id: String) -> void  # SaveManager에 저장
func get_unlocked_list() -> Array[ImagenData]
```

### 수락 기준

- [ ] ImagenDatabase가 JSON에서 이마젠 데이터를 로드한다

- [ ] `get_unlocked_list()`가 해금된 이마젠만 반환한다
- [ ] 속성별 색상이 올바르게 반환된다

### 의존성

### 구현 노트

## Task 2: SkillManager

### 설명
이마젠 스킬의 발동과 쿨타임을 관리하는 매니저. 스킬 발동 시 턴을 소모하지 않으며, 쿨타임이 끝날 때까지 재사용 불가.

### 파일 경로
- `scripts/companion/skill_manager.gd`

### 상세 구현

#### 핵심 API
```gdscript
class_name SkillManager
extends Node

signal skill_activated(skill_id: String)
signal skill_cooldown_updated(slot: int, remaining: int)
signal skill_ready(slot: int)
signal skill_requires_target(skill_id: String, target_type: String)
signal skill_target_selected(skill_id: String, target_data: Variant)

# 편성된 이마젠 스킬 슬롯 (최대 2개)
var skill_slots: Array[Dictionary] = []
# 각 슬롯: { imagen_id, skill_id, cooldown_max, cooldown_remaining, is_ready }

func setup_party(imagenes: Array[ImagenData]) -> void:
    # 편성된 이마젠의 스킬로 슬롯 초기화
    # cooldown_remaining = 0 (즉시 사용 가능)

func activate_skill(slot: int) -> bool:
    # 쿨타임 확인 → 타겟 필요 시 타겟 선택 요청 → 스킬 실행
    # 턴 소모 없음
    # 성공 시 쿨타임 설정, true 반환

func on_turn_end() -> void:
    # 모든 슬롯의 cooldown_remaining -= 1
    # 0이 되면 skill_ready 시그널 발행

func get_slot_info(slot: int) -> Dictionary:
    # UI 표시용 슬롯 정보 반환

func is_skill_ready(slot: int) -> bool
```

#### 쿨타임 규칙
- 스킬 사용 직후 `cooldown_remaining = cooldown_max`로 설정
- 매 턴 종료 시 `cooldown_remaining -= 1` (턴 = 플레이어의 유효한 색칠 액션 1회)
- `cooldown_remaining == 0`이면 사용 가능
- 인피니티 모드: 턴 대신 "유효 액션 횟수" 기준으로 쿨타임 감소 (동일 로직)

#### 타겟 선택 시스템
스킬에 따라 타겟 선택이 필요한 경우:
```gdscript
enum TargetType {
    NONE,           # 타겟 불필요 (K6, K7, K8, K9, K10, K11)
    COLOR,          # 색상 선택 (K1, K3, K4)
    COLOR_PAIR,     # 두 색상 선택 (K3)
    CELL,           # 셀 선택 (K2)
    ROW_OR_COL,     # 행 또는 열 선택 (K5)
}
```

- 타겟이 필요한 스킬: `skill_requires_target` 시그널 발행 → UI에서 타겟 선택 모드 진입 → 선택 완료 시 `skill_target_selected` 시그널 → 스킬 실행
- 타겟 선택 취소 시 스킬 미발동, 쿨타임 미적용

#### 스킬 발동 → GameManager 연결
```gdscript
# SkillManager가 스킬 실행 시 GameManager에 알림
# GameManager는 그리드 상태 변경 후 연쇄 체크 실행

signal request_grid_action(action: Dictionary)
# action 예시: { "type": "recolor", "positions": [...], "color": 2 }
# action 예시: { "type": "destroy", "positions": [...] }
# action 예시: { "type": "set_board", "board_state": [...] }
```

### 수락 기준
- [ ] 편성된 이마젠의 스킬이 슬롯에 등록된다
- [ ] 스킬 발동 시 턴이 소모되지 않는다
- [ ] 발동 후 쿨타임이 설정되고, 매 턴 감소한다
- [ ] 쿨타임 중인 스킬은 발동 불가하다
- [ ] 타겟 필요 스킬은 타겟 선택 과정을 거친다
- [ ] 타겟 선택 취소 시 스킬이 소비되지 않는다
- [ ] `skill_cooldown_updated` 시그널이 매 턴 발행된다

### 의존성
- Task 1 (ImagenData)
- S03 GameManager/TurnManager

### 구현 노트
- SkillManager는 GameManager의 자식 노드로 배치하거나 Autoload로 등록
- TurnManager의 `turn_ended` 시그널에 `on_turn_end()` 연결
- 스킬 실행 로직 자체는 개별 스킬 스크립트(Task 3~5)에 위임
- 스킬 발동 중에는 다른 입력(셀 터치, 다른 스킬)을 블록해야 함

### K1: 단색폭풍 (Color Storm)

#### 효과
필드 위 지정 색상 블록 N개를 다른 색으로 변환한다.

#### 파라미터
- `N`: 변환 블록 수 (기본 5, 이마젠별 조정 가능)

#### 타겟 선택
- **TargetType: COLOR_PAIR** (순차 2단계)
  1. 원본 색상 선택: 화면에 현재 보드에 존재하는 색상 팔레트 표시 → 탭
  2. 변환할 색상 선택: 원본을 제외한 팔레트 표시 → 탭

#### 실행 로직
```
1. 보드에서 원본 색상인 셀 목록 수집 (메인 영역만, 버퍼 제외)
2. 셀 목록에서 랜덤 N개 선택 (N보다 적으면 전부)
3. 선택된 셀의 색상을 변환 색상으로 변경
4. 변경 후 라인 완성 체크 → 연쇄 콤보 트리거
```

#### 그리드 상호작용
- 잠긴 칸(M1): 색 변경 불가 → 대상에서 제외
- 돌 칸(M2): 색 없음 → 대상에서 제외
- 얼음 칸(M3): 얼음 안의 색도 변환 대상 (얼음은 유지)
- 무지개 칸(M4): 모든 색에 매칭 → 원본 색 선택 시 포함. 색 변환 시 무지개 유지
- 퇴색 칸(S3): 변환 가능하지만 2턴 후 원래 색으로 복귀 (퇴색 타이머 리셋)

#### 엣지 케이스
- 원본 색이 보드에 없는 경우: `can_use` 단계에서 확인 불필요 (팔레트에 표시된 색만 선택 가능)
- N이 해당 색 총 개수보다 많은 경우: 존재하는 만큼만 변환
- 변환 결과 라인 완성 시 연쇄 콤보 정상 진행

#### 이펙트 연출
- 선택된 셀들이 순차적으로 색상 전환 (Tween 0.1초 간격)
- 전환 시 파티클 이펙트 (작은 별 모양 GPUParticles2D)

### 수락 기준 (K1)
- [ ] 두 색상을 순차 선택할 수 있다
- [ ] N개 블록이 올바르게 변환된다
- [ ] 잠긴/돌 칸은 변환 대상에서 제외된다
- [ ] 변환 후 라인 완성 시 연쇄 콤보가 동작한다
- [ ] 변환 애니메이션이 재생된다

### K2: 무지개파동 (Rainbow Wave)

#### 효과
지정 셀 기준 3×3 영역 내 모든 칸을 선택한 색으로 통일한다.

#### 파라미터
- 영역 크기: 3×3 (고정)

#### 타겟 선택
- **TargetType: CELL + COLOR** (2단계)
  1. 중심 셀 선택: 보드에서 셀 탭 (3×3 영역 프리뷰 표시)
  2. 통일할 색상 선택: 팔레트 표시 → 탭

#### 실행 로직
```
1. 중심 셀 좌표 (cx, cy) 기준 3×3 영역 계산
   범위: (cx-1, cy-1) ~ (cx+1, cy+1), 보드 경계 클램프
2. 영역 내 변경 가능한 셀을 선택한 색으로 변경
3. 변경 후 라인 완성 체크 → 연쇄 콤보 트리거
```

#### 그리드 상호작용
- 잠긴 칸(M1): 변경 불가 → 건너뜀
- 돌 칸(M2): 건너뜀
- 얼음 칸(M3): 얼음 안의 색 변경, 얼음 유지
- 앵커 칸(M5): 색 변경 가능 (앵커 속성 유지)
- 페인트통(M6): 색 변경 시 페인트통 효과 발동 (행/열 전체 변환)

#### 엣지 케이스
- 보드 구석/변에서 사용: 3×3 영역이 보드 밖으로 나가면 보드 내 영역만 적용
- 버퍼 영역: 메인 그리드만 대상, 버퍼 제외
- 페인트통이 3×3 안에 있으면: 페인트통 효과가 추가 트리거되어 대규모 변환 가능 (의도된 시너지)

#### 이펙트 연출
- 중심에서 3×3으로 파동 퍼짐 효과 (Tween scale 1→3×3)
- 각 셀 색 전환 시 반짝임

#### 타겟 선택 UI
- 셀 선택 모드 진입 시 보드 위에 3×3 그리드 오버레이 표시
- 손가락/마우스 이동에 따라 오버레이가 따라다님
- 유효한 영역이면 초록 테두리, 무효(전부 돌/잠긴)이면 빨간 테두리

### 수락 기준 (K2)
- [ ] 셀 탭 시 3×3 영역 프리뷰가 표시된다
- [ ] 선택한 색으로 3×3 영역이 통일된다
- [ ] 잠긴/돌 칸은 건너뛴다
- [ ] 보드 경계에서 클램프 처리된다
- [ ] 페인트통 시너지가 동작한다
- [ ] 파동 이펙트가 재생된다

### K3: 컬러스왑 (Color Swap)

#### 효과
필드 전체에서 두 색상을 서로 교환한다.

#### 타겟 선택
- **TargetType: COLOR_PAIR**
  1. 첫 번째 색상 선택: 팔레트 → 탭
  2. 두 번째 색상 선택: 첫 번째 제외 팔레트 → 탭

#### 실행 로직
```
1. 보드 전체(메인+버퍼) 순회
2. 색상 A인 셀 → 색상 B로 변경
3. 색상 B인 셀 → 색상 A로 변경
   (동시 처리: 먼저 전체 탐색 후 일괄 변경)
4. 변경 후 라인 완성 체크 → 연쇄 콤보 트리거
```

#### 그리드 상호작용
- 잠긴 칸(M1): 스왑 대상에서 제외
- 돌 칸(M2): 색 없음, 무관
- 얼음 칸(M3): 얼음 안의 색도 스왑 대상
- 무지개 칸(M4): 색이 없으므로 스왑 무관 (무지개 유지)
- 페인트통(M6): 스왑으로 색이 바뀌면 페인트통 효과는 발동하지 않음 (직접 칠하기만 발동)
- 퇴색 칸(S3): 스왑 가능, 복귀 색도 함께 스왑됨 (원래 색 A → 원래 색 B)
- 버퍼 영역: 버퍼의 색상도 스왑 대상

#### 엣지 케이스

### K6: 되감기 (Undo)

#### 효과
직전 1회 액션의 보드 상태로 복구한다.

#### 타겟 선택
- **TargetType: NONE**

#### 실행 로직
```
1. 스냅샷 스택에서 직전 상태 pop
2. 그리드 상태 복원 (모든 셀의 색상 + 기믹 상태)
3. 컬러 큐 상태 복원 (이전 큐 포인터)
4. 턴 카운트 복원 (+1턴 복구)
5. 파괴 카운트 복원
6. 렌더링 갱신
```

#### 스냅샷 시스템
```gdscript
# GameManager에 추가
var board_snapshots: Array[Dictionary] = []
const MAX_SNAPSHOTS := 10

func save_snapshot() -> void:
    var snapshot := {
        "grid": grid.serialize(),         # 전체 셀 상태 딕셔너리
        "queue_index": color_queue.current_index,
        "turns_remaining": turn_manager.turns_remaining,
        "destroy_count": destroy_count,
        "gimmick_states": _serialize_gimmick_states()
    }
    board_snapshots.push_back(snapshot)
    if board_snapshots.size() > MAX_SNAPSHOTS:
        board_snapshots.pop_front()

func restore_snapshot() -> bool:
    if board_snapshots.is_empty():
        return false
    var snapshot = board_snapshots.pop_back()
    grid.deserialize(snapshot.grid)
    color_queue.current_index = snapshot.queue_index
    turn_manager.turns_remaining = snapshot.turns_remaining
    destroy_count = snapshot.destroy_count
    _restore_gimmick_states(snapshot.gimmick_states)
    return true
```

#### 그리드 상호작용
- 모든 기믹 상태도 함께 복원됨 (얼음 내구도, 퇴색 카운터, 번짐 상태 등)
- 파괴된 셀도 복원됨

#### 엣지 케이스
- 스냅샷이 없는 경우 (첫 턴): `can_use` → false 반환, 버튼 비활성
- 되감기 후 다시 되감기: 2단계 이전으로 복구 (스냅샷이 남아 있으면)
- 되감기 자체는 스냅샷에 저장하지 않음 (되감기의 되감기 방지를 원하면 스냅샷 저장 스킵)
- 스킬로 인한 변경도 스냅샷에 포함 (스킬 사용 전에도 스냅샷 저장)

#### 이펙트 연출
- 화면 전체 역재생 느낌의 효과 (보드가 잠시 반투명 → 이전 상태로 전환)

### K7: 미래의눈 (Future's Eye)

- 잠긴 칸: 리컬러 불가이므로 제외
- 돌 칸: 터치 불가이므로 제외

#### 이펙트 연출

```gdscript

var is_timer_paused: bool = false
var pause_remaining: float = 0.0

func pause_timer(duration: float) -> void:
    is_timer_paused = true
    pause_remaining = duration

func _process(delta: float) -> void:

        pause_remaining -= delta
        if pause_remaining <= 0:
            is_timer_paused = false
        return  # 타이머 감소 스킵
```

## Task 10: PartyManager (`party_manager.gd`)

### 설명
편성(파티) 상태를 관리하는 싱글턴. 스테이지 시작 전 선택된 이마젠 슬롯 정보를 저장하고, 게임 씬과 편성 UI 양쪽에서 동일한 상태를 참조할 수 있도록 중간 매니저 역할을 한다.

> Task 6(편성 화면 UI)이 내부 로직까지 담당할 수도 있지만, 편성 상태를 씬 전환 후에도 유지해야 하므로 별도 Autoload로 분리한다.

### 파일 경로
- `scripts/companion/party_manager.gd`

### 상세 구현

```gdscript
extends Node  # Autoload singleton

const MAX_PARTY_SIZE = 2

var selected_party: Array[String] = []  # imagen_id 배열, 최대 2개

signal party_changed(party: Array[String])

func set_party(imagen_ids: Array[String]) -> void:
    selected_party = imagen_ids.slice(0, MAX_PARTY_SIZE)
    emit_signal("party_changed", selected_party)
    # SaveManager에 저장 (마지막 편성 기억)
    SaveManager.save_last_party(selected_party)

func get_party() -> Array[ImagenData]:
    var result: Array[ImagenData] = []
    for id in selected_party:
        var data = ImagenDatabase.get_imagen(id)
        if data:
            result.append(data)
    return result

func load_saved_party() -> void:
    # 앱 재시작 시 마지막 편성 복원
    selected_party = SaveManager.get_last_party()
```

### 수락 기준
- [ ] `set_party(["fire_bulkkot", "water_mulgyeol"])`로 2마리 편성이 저장된다
- [ ] `get_party()`가 ImagenData 배열을 반환한다
- [ ] 3마리 이상 전달 시 MAX_PARTY_SIZE(2)로 잘린다
- [ ] 앱 재시작 후 마지막 편성이 복원된다
- [ ] `party_changed` 시그널이 UI에 반영된다
