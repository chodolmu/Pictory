## Task 1: 로고 / 스플래시 화면

### 설명

앱 실행 시 가장 먼저 표시되는 스플래시 화면을 구현한다. 로고(텍스트 기반)가 페이드 인 → 유지 → 페이드 아웃되고 자동으로 타이틀 화면으로 전환된다.

### UI 구조 (씬 트리)

```
SplashScreen (Control)
  ├─ ColorRect (배경 — 단색, 예: 흰색 또는 검정)
  └─ CenterContainer
      └─ LogoLabel (Label 또는 RichTextLabel)
          - text: "PICTORY" 또는 팀/퍼블리셔 로고 텍스트
          - 큰 폰트, 중앙 정렬
```

### 상세 구현

```gdscript
class_name SplashScreen
extends Control

@export var fade_in_duration: float = 0.8
@export var hold_duration: float = 1.5
@export var fade_out_duration: float = 0.8

func _ready() -> void:
    _logo_label.modulate.a = 0.0
    _play_sequence()

func _play_sequence() -> void:
    var tween = create_tween()
    tween.tween_property(_logo_label, "modulate:a", 1.0, fade_in_duration)
    tween.tween_interval(hold_duration)
    tween.tween_property(_logo_label, "modulate:a", 0.0, fade_out_duration)
    tween.tween_callback(_go_to_title)

func _go_to_title() -> void:
    SceneManager.change_scene("res://scenes/main/title.tscn")

func _input(event: InputEvent) -> void:
    # 터치/클릭 시 스킵
    if event is InputEventMouseButton and event.pressed:
        _go_to_title()
```

### Acceptance Criteria

- [ ] 앱 실행 시 스플래시 화면이 가장 먼저 표시된다
- [ ] 로고 텍스트가 페이드 인(0.8초) → 유지(1.5초) → 페이드 아웃(0.8초) 순서로 진행된다
- [ ] 페이드 아웃 완료 후 자동으로 타이틀 화면으로 전환된다
- [ ] 화면 터치/클릭으로 스킵이 가능하다
- [ ] 스킵 시에도 SceneManager를 통한 정상 전환이 수행된다

### Dependencies

- Task 9 (SceneManager — 씬 전환)
- 단, SceneManager 없이도 `get_tree().change_scene_to_file()` 임시 사용 가능

### 구현 노트

- **프로젝트 메인 씬**: `Project > Project Settings > Application > Run > Main Scene`을 `scenes/main/splash.tscn`으로 설정
- Tween 체이닝: `create_tween()` → `tween_property()` → `tween_interval()` → `tween_property()` → `tween_callback()` 순차 실행
- AI-only 원칙: 로고는 Label 노드에 큰 폰트 + `modulate` 색상으로 구현. 이미지 사용 불가.
- 폰트 크기: `theme_override_font_sizes/font_size = 72` 이상

### 파일

| 파일 | 타입 | 설명 |
|------|------|------|
| `scenes/main/splash.tscn` | Scene | 스플래시 화면 씬 |
| `scripts/main/splash_screen.gd` | GDScript | 스플래시 화면 로직 |

## Task 2: 타이틀 화면

### 설명

로고 후 표시되는 타이틀 화면을 구현한다. 게임 로고(큰 텍스트)와 [게임 시작] 버튼을 배치하고, 버튼 클릭 시 메인 화면으로 이동한다.

### UI 구조 (씬 트리)

```
TitleScreen (Control)
  ├─ ColorRect (배경)
  └─ VBoxContainer (중앙 정렬)
      ├─ GameTitleLabel ("PICTORY")
      │   - 큰 폰트, 중앙 정렬
      │   - Tween: 부드러운 반복 scale 펄스 (0.98~1.02) 또는 색상 그라데이션
      ├─ MarginContainer (간격)
      └─ StartButton (Button)
          - text: "게임 시작"
          - 중앙 정렬, 충분한 크기 (모바일 터치 최소 48dp)
```

### 상세 구현

```gdscript
class_name TitleScreen
extends Control

func _ready() -> void:
    _start_button.pressed.connect(_on_start_pressed)
    _animate_title()

func _animate_title() -> void:
    var tween = create_tween().set_loops()
    tween.tween_property(_title_label, "scale", Vector2(1.02, 1.02), 1.0) \
         .set_trans(Tween.TRANS_SINE)
    tween.tween_property(_title_label, "scale", Vector2(0.98, 0.98), 1.0) \
         .set_trans(Tween.TRANS_SINE)

func _on_start_pressed() -> void:
    # 최초 접속 시 닉네임 입력 화면으로, 이미 닉네임 설정된 경우 메인 메뉴로
    if PlayerProfile.is_first_launch():
        SceneManager.change_scene("res://scenes/main/nickname_input.tscn")
    else:
        SceneManager.change_scene("res://scenes/main/main_menu.tscn")
```

### Acceptance Criteria

- [ ] 타이틀 화면에 "PICTORY" 게임 로고가 표시된다
- [ ] 로고에 부드러운 반복 애니메이션(scale 펄스)이 적용된다
- [ ] [게임 시작] 버튼이 화면 하단 중앙에 위치한다
- [ ] [게임 시작] 클릭 시 최초 접속이면 닉네임 입력 화면으로, 아니면 메인 화면으로 전환된다
- [ ] 버튼 크기가 모바일 터치에 적합하다 (최소 48x48 dp)

### Dependencies

- 배경색: 게임 분위기에 맞는 단색 또는 그라데이션 (ColorRect + shader 또는 두 ColorRect 겹침)

## Task 2-B: 최초 접속 닉네임 입력 화면

### 설명

최초 실행 시(닉네임 미설정 상태) 타이틀 화면에서 [게임 시작]을 누르면 닉네임 입력 화면으로 이동한다. 닉네임을 입력·확인하면 PlayerProfile에 저장되고 메인 메뉴로 진입한다. 이후 실행부터는 이 화면을 건너뛴다.

### UI 구조 (씬 트리)

```
NicknameInput (Control)
  ├─ ColorRect (배경)
  └─ VBoxContainer (중앙 정렬)
      ├─ PromptLabel ("닉네임을 입력하세요")
      ├─ NicknameLineEdit (LineEdit — max_length: 12)
      ├─ ValidationLabel ("2~12자로 입력해주세요" — 조건 미충족 시 빨간색)
      └─ ConfirmButton (Button — "확인")
```

### 상세 구현

```gdscript
class_name NicknameInputScreen
extends Control

func _ready() -> void:
    _confirm_btn.pressed.connect(_on_confirm)
    _nickname_edit.text_changed.connect(_on_text_changed)
    _confirm_btn.disabled = true

func _on_text_changed(new_text: String) -> void:
    var valid = new_text.strip_edges().length() >= 2 and new_text.strip_edges().length() <= 12
    _confirm_btn.disabled = not valid
    _validation_label.text = "" if valid else "2~12자로 입력해주세요"

func _on_confirm() -> void:
    var nickname = _nickname_edit.text.strip_edges()
    PlayerProfile.set_nickname(nickname)
    PlayerProfile.set_first_launch(false)
    SceneManager.change_scene("res://scenes/main/main_menu.tscn")
```

### Acceptance Criteria

- [ ] 최초 실행 시(PlayerProfile.is_first_launch() == true) 타이틀 → 닉네임 입력 화면으로 이동
- [ ] LineEdit에 2~12자 입력 시 [확인] 버튼 활성화
- [ ] 1자 이하 입력 시 [확인] 비활성 + 안내 메시지 표시
- [ ] [확인] 클릭 → PlayerProfile에 닉네임 저장 + first_launch 플래그 해제 → 메인 메뉴로 전환
- [ ] 이후 실행부터는 닉네임 입력 화면을 건너뛰고 바로 메인 메뉴로 이동

### Dependencies

- Task 2 (타이틀 화면 — 분기 로직)
- Task 10-B (PlayerProfile)
- Task 9 (SceneManager)

### 파일

| 파일 | 타입 | 설명 |
|------|------|------|

게임의 중앙 허브 화면을 구현한다. UI 플로우 다이어그램의 메인 화면에 해당하며, 모든 주요 기능으로의 진입점을 제공한다. 업적/컬렉션/상점 버튼은 S10/S11에서 연결하므로 placeholder 상태로 배치한다.

```
MainMenu (Control)
  ├─ ColorRect (배경)

  │   │   │   ├─ PlayerIconRect (ColorRect/TextureRect — 선택된 플레이어 아이콘)

  │   │   ├─ CurrencyLabel ("코인: 1,500")  # SaveManager 연동

  │   │   ├─ AchievementsButton (Button — "업적") [placeholder]
  │   │   ├─ CollectionButton (Button — "컬렉션") [placeholder]
```

### 상세 구현

```gdscript
class_name MainMenu
extends Control

func _ready() -> void:
    _stage_select_btn.pressed.connect(_on_stage_select)
    _infinity_btn.pressed.connect(_on_infinity)
    _settings_btn.pressed.connect(_on_settings)
    _achievements_btn.pressed.connect(_on_placeholder.bind("업적"))
    _collection_btn.pressed.connect(_on_placeholder.bind("컬렉션"))
    _shop_btn.pressed.connect(_on_placeholder.bind("상점"))
    _update_currency_display()
    _update_player_info()

func _on_stage_select() -> void:

func _on_infinity() -> void:
    _infinity_confirm_popup.show_popup()

func _on_settings() -> void:
    _options_popup.show_popup()

func _on_placeholder(feature_name: String) -> void:
    print("[placeholder] %s — S10/S11에서 구현 예정" % feature_name)
    # 임시: 비활성 토스트 메시지 또는 아무 동작 없음

func _update_currency_display() -> void:
    _currency_label.text = "코인: %s" % SaveManager.get_currency()

func _update_player_info() -> void:
    _nickname_label.text = PlayerProfile.get_nickname()
    # 플레이어 아이콘은 PlayerProfile.get_selected_icon()으로 설정

func _notification(what: int) -> void:
    if what == NOTIFICATION_WM_GO_BACK_REQUEST:
        _confirm_quit_popup.show_popup()
```

### Acceptance Criteria

- [ ] 좌측 상단에 플레이어 닉네임 + 아이콘이 표시된다 (PlayerProfile 연동)
- [ ] 메인 화면에 [스테이지 선택], [인피니티 모드], [업적], [컬렉션], [옵션], [상점] 버튼이 모두 표시된다
- [ ] [스테이지 선택] 클릭 → 스테이지 선택 화면으로 이동한다
- [ ] [인피니티 모드] 클릭 → 인피니티 확인 팝업이 표시된다
- [ ] [옵션] 클릭 → 옵션 팝업이 표시된다
- [ ] [업적], [컬렉션], [상점] 클릭 시 placeholder 동작(콘솔 로그 또는 "준비 중" 알림)이 실행된다
- [ ] 현재 보유 재화가 상단에 표시된다
- [ ] Android 뒤로가기(또는 ESC 키) → 게임 종료 확인 팝업이 표시된다
- [ ] [챕터 이동]은 스테이지 선택 화면 내 탭으로 통합 처리한다

### Dependencies

- Task 10 (SaveManager — 재화 표시)

### 구현 노트

- **뒤로가기 처리**: Godot 4.x에서 `NOTIFICATION_WM_GO_BACK_REQUEST`는 Android 백버튼 + 프로젝트 설정에서 `input_devices/pointing/emulate_touch_from_mouse` 활성화 시 동작. PC에서는 ESC 키를 `_unhandled_input()`에서 별도 감지
- [챕터 이동]은 별도 화면 대신 StageSelect 화면의 챕터 탭으로 통합. 메인 화면 버튼은 "스테이지 선택" 하나로 충분 (챕터 내비게이션은 StageSelect 내에서)
- Placeholder 버튼: `disabled = false`로 두되 클릭 시 "준비 중" 토스트 또는 콘솔 로그. S10/S11에서 실제 씬 연결
- 버튼 레이아웃: 주요 버튼(스테이지/인피니티)은 크게, 보조 버튼(업적/컬렉션/상점)은 하단에 작게 배치
- 재화 표시: `SaveManager.get_currency()` 호출, `_ready()`와 화면 복귀(`_on_visibility_changed`) 시 갱신

### 상세 구현

```gdscript
class_name StageSelectScreen
extends Control

var current_chapter: int = 1
var stage_buttons: Array[StageButton] = []

func _ready() -> void:
    _back_btn.pressed.connect(_on_back)
    _load_chapter(current_chapter)
    _setup_chapter_tabs()

func _load_chapter(chapter: int) -> void:
    current_chapter = chapter
    var stages = LevelLoader.load_chapter_stages(chapter)
    _clear_stage_grid()
    for config in stages:
        var btn = _create_stage_button(config)
        _stage_grid.add_child(btn)
        stage_buttons.append(btn)
    _update_chapter_tabs()
    _update_unlock_info()

func _create_stage_button(config: StageConfig) -> StageButton:
```

## Task 9: 씬 전환 매니저 (SceneManager)

### 설명

모든 씬 간 전환을 관리하는 Autoload 싱글턴을 구현한다. 페이드 아웃 → 씬 교체 → 페이드 인 트랜지션을 제공하고, 전환 시 파라미터를 전달할 수 있다.

### 상세 구현

```gdscript
extends CanvasLayer  # Autoload singleton, layer = 100 (최상위)

signal scene_changed(scene_path: String)

@export var fade_duration: float = 0.3
@export var fade_color: Color = Color.BLACK

var _transition_params: Dictionary = {}
var _color_rect: ColorRect
var _is_transitioning: bool = false

func _ready() -> void:
    layer = 100
    _setup_fade_overlay()

func _setup_fade_overlay() -> void:
    _color_rect = ColorRect.new()
    _color_rect.color = fade_color
    _color_rect.mouse_filter = Control.MOUSE_FILTER_IGNORE
    _color_rect.set_anchors_and_offsets_preset(Control.PRESET_FULL_RECT)
    _color_rect.modulate.a = 0.0
    add_child(_color_rect)

func change_scene(scene_path: String, params: Dictionary = {}) -> void:
    if _is_transitioning:
        return
    _is_transitioning = true
    _transition_params = params
    var tween = create_tween()
    _color_rect.mouse_filter = Control.MOUSE_FILTER_STOP  # 전환 중 입력 차단
    tween.tween_property(_color_rect, "modulate:a", 1.0, fade_duration)
    tween.tween_callback(_do_change_scene.bind(scene_path))
    tween.tween_property(_color_rect, "modulate:a", 0.0, fade_duration)
    tween.tween_callback(_on_transition_complete)

func _do_change_scene(scene_path: String) -> void:
    get_tree().change_scene_to_file(scene_path)
    scene_changed.emit(scene_path)

func _on_transition_complete() -> void:
    _color_rect.mouse_filter = Control.MOUSE_FILTER_IGNORE
    _is_transitioning = false

func get_params() -> Dictionary:
    return _transition_params

func get_param(key: String, default = null):
    return _transition_params.get(key, default)
```

- [ ] 페이드 색상과 지속시간이 `@export`로 조절 가능하다
- [ ] CanvasLayer `layer = 100`으로 모든 UI 위에 페이드 오버레이가 표시된다

### 상세 구현

```gdscript
extends Node  # Autoload singleton

const SAVE_PATH := "user://save_data.json"
const SAVE_VERSION := 1

var _data: Dictionary = {}

func _ready() -> void:
    _load()

# --- Stage Data ---
func get_stage_data(stage_id: String) -> Dictionary:
    # Returns {"cleared": bool, "stars": int, "best_score": int} or empty dict
    return _data.get("stages", {}).get(stage_id, {})

func save_stage_result(stage_id: String, stars: int, score: int) -> void:
    if not _data.has("stages"):
        _data["stages"] = {}
    var existing = _data["stages"].get(stage_id, {"cleared": false, "stars": 0, "best_score": 0})
    _data["stages"][stage_id] = {
        "cleared": true,
        "stars": max(existing.get("stars", 0), stars),  # 최고 별 수 유지
        "best_score": max(existing.get("best_score", 0), score)
    }
    _save()

func is_stage_cleared(stage_id: String) -> bool:
    return get_stage_data(stage_id).get("cleared", false)

func get_stage_stars(stage_id: String) -> int:
    return get_stage_data(stage_id).get("stars", 0)

# --- Currency ---
func get_currency() -> int:
    return _data.get("currency", 0)

func add_currency(amount: int) -> void:
    _data["currency"] = get_currency() + amount
    _save()

func spend_currency(amount: int) -> bool:
    if get_currency() >= amount:
        _data["currency"] = get_currency() - amount
        _save()
        return true
    return false

# --- Chapter Unlock ---
func get_unlocked_chapters() -> Array:
    return _data.get("chapters_unlocked", [1])

func is_chapter_unlocked(chapter: int) -> bool:
    return chapter in get_unlocked_chapters()

func unlock_chapter(chapter: int) -> void:
    var unlocked = get_unlocked_chapters()
    if chapter not in unlocked:
        unlocked.append(chapter)
        _data["chapters_unlocked"] = unlocked
        _save()

# --- Infinity Mode ---
func get_infinity_high_score() -> int:
    return _data.get("infinity", {}).get("high_score", 0)

func save_infinity_result(score: int) -> bool:
    # Returns true if new high score
    if not _data.has("infinity"):
        _data["infinity"] = {"high_score": 0, "total_plays": 0}
    _data["infinity"]["total_plays"] += 1
    var is_new_record = score > _data["infinity"]["high_score"]
    if is_new_record:
        _data["infinity"]["high_score"] = score
    _save()
    return is_new_record

# --- Settings ---
func get_settings() -> Dictionary:
    return _data.get("settings", {"bgm_volume": 80, "se_volume": 80})

func save_settings(settings: Dictionary) -> void:
    _data["settings"] = settings
    _save()

# --- Internal ---
func _load() -> void:
    if not FileAccess.file_exists(SAVE_PATH):
        _data = _create_default_data()
        _save()
        return
    var file = FileAccess.open(SAVE_PATH, FileAccess.READ)
    var text = file.get_as_text()
    file.close()
    var parsed = JSON.parse_string(text)
    if parsed is Dictionary:
        _data = parsed
        _migrate_if_needed()
    else:
        push_error("SaveManager: 세이브 파일 파싱 실패, 기본값으로 초기화")
        _data = _create_default_data()
        _save()

func _save() -> void:
    var file = FileAccess.open(SAVE_PATH, FileAccess.WRITE)
    file.store_string(JSON.stringify(_data, "\t"))
    file.close()

func _create_default_data() -> Dictionary:
    return {
        "version": SAVE_VERSION,
        "currency": 0,
        "stages": {},
        "chapters_unlocked": [1],
        "infinity": {"high_score": 0, "total_plays": 0},
        "settings": {"bgm_volume": 80, "se_volume": 80}
    }

func _migrate_if_needed() -> void:
    var file_version = _data.get("version", 0)
    if file_version < SAVE_VERSION:
        # 버전별 마이그레이션 로직 추가
        _data["version"] = SAVE_VERSION
        _save()

func reset_all_data() -> void:
    _data = _create_default_data()
    _save()
```

### Acceptance Criteria

- [ ] 앱 첫 실행 시 기본 세이브 파일이 `user://save_data.json`에 생성된다

## Task 11: 챕터 해금 로직

### 설명

재화(코인)를 사용하여 다음 챕터를 해금하는 시스템을 구현한다. **챕터 단위로 해금**하며, 해금된 챕터 내의 스테이지는 순차적으로 플레이할 수 있다 (스테이지별 비용 없음). 스테이지 선택 화면(Task 4)과 SaveManager(Task 10)를 연동한다.

### 해금 모델

- **챕터 해금**: 재화로 챕터를 구매하여 해금한다. 이전 챕터의 올클리어가 선행 조건이다.
- **스테이지 진행**: 해금된 챕터 내에서 스테이지는 순차적으로 해금된다 (이전 스테이지 클리어 시 다음 스테이지 자동 해금). 스테이지별 재화 소모는 없다.

### 해금 조건

| 챕터 | 해금 조건 |
|------|-----------|
| 1 | 기본 해금 (무료) |
| 2 | 챕터 1 올클리어 + 500 코인 |
| 3 | 챕터 2 올클리어 + 800 코인 |
| 4 | 챕터 3 올클리어 + 1200 코인 |
| 5~10 | 이전 챕터 올클리어 + 점진 증가 코인 |

### 상세 구현

```gdscript
# chapter_unlock.gd — LevelLoader에 통합하거나 별도 유틸리티
class_name ChapterUnlock
extends RefCounted

const CHAPTER_COSTS: Dictionary = {
    2: 500,
    3: 800,
    4: 1200,
    5: 1600,
    6: 2000,
    7: 2500,
    8: 3000,
    9: 3500,
    10: 4000
}

static func get_unlock_cost(chapter: int) -> int:
    return CHAPTER_COSTS.get(chapter, 0)

static func can_unlock(chapter: int) -> Dictionary:
    # Returns {"can_unlock": bool, "reason": String, "cost": int}
    var cost = get_unlock_cost(chapter)
    var prev_chapter = chapter - 1

    # 이전 챕터 올클리어 확인
    if not _is_chapter_all_cleared(prev_chapter):
        return {
            "can_unlock": false,
            "reason": "챕터 %d을 모두 클리어해야 합니다" % prev_chapter,
            "cost": cost
        }

    # 재화 확인
    if SaveManager.get_currency() < cost:
        return {
            "can_unlock": false,
            "reason": "코인이 부족합니다 (%d/%d)" % [SaveManager.get_currency(), cost],
            "cost": cost
        }

    return {"can_unlock": true, "reason": "", "cost": cost}

static func try_unlock(chapter: int) -> bool:
    var check = can_unlock(chapter)
    if not check["can_unlock"]:
        return false
    SaveManager.spend_currency(check["cost"])
    SaveManager.unlock_chapter(chapter)
    return true

static func _is_chapter_all_cleared(chapter: int) -> bool:
    for i in range(1, 11):  # 10 stages per chapter
        var stage_id = "ch%02d_s%02d" % [chapter, i]
        if not SaveManager.is_stage_cleared(stage_id):
            return false
    return true
```

```gdscript
    _unlock_info_label.visible = true

        _unlock_info_label.text = "다음 챕터 해금 가능! (%d 코인)" % check["cost"]
        _unlock_button.visible = true
    else:
        _unlock_info_label.text = check["reason"]
        _unlock_button.visible = false

func _on_unlock_button_pressed() -> void:
    var next_chapter = current_chapter + 1
    if ChapterUnlock.try_unlock(next_chapter):
        _load_chapter(next_chapter)
        _update_chapter_tabs()
```

### Acceptance Criteria

- [ ] `ChapterUnlock.can_unlock(2)` — 챕터 1 미올클리어 시 `false` + 사유 메시지
- [ ] 챕터 1 올클리어 + 재화 500 이상 → `try_unlock(2)` 성공, 재화 500 차감, 챕터 2 해금
- [ ] 재화 부족 시 `try_unlock()` 실패, 재화/해금 상태 변경 없음
- [ ] 스테이지 선택 화면에 해금 비용 + 조건이 표시된다
- [ ] 해금 가능 시 [해금] 버튼이 표시되고, 클릭 시 즉시 챕터 전환된다

## Task 12: 인게임 뒤로가기

### 설명

```
      ├─ DimOverlay

              ├─ MessageLabel ("메인 화면으로 돌아가시겠습니까?\n진행 상황이 저장되지 않습니다.")
              ├─ HSeparator
              └─ ButtonContainer (HBoxContainer)
                  ├─ ConfirmButton ("확인")
                  └─ CancelButton ("취소")
```

### 상세 구현

```gdscript
# game_manager.gd에 추가 또는 별도 스크립트
func _on_back_button_pressed() -> void:
    _pause_game()
    _confirm_exit_popup.visible = true

func _pause_game() -> void:
    get_tree().paused = true  # 또는 모드별 타이머 일시정지

func _resume_game() -> void:
    get_tree().paused = false

func _on_confirm_exit() -> void:
    get_tree().paused = false
    SceneManager.change_scene("res://scenes/main/main_menu.tscn")

func _on_cancel_exit() -> void:
    _confirm_exit_popup.visible = false
    _resume_game()

func _notification(what: int) -> void:
    if what == NOTIFICATION_WM_GO_BACK_REQUEST:
        if _confirm_exit_popup.visible:
            _on_cancel_exit()
        else:
            _on_back_button_pressed()
```

### Acceptance Criteria

- [ ] Android 뒤로가기를 팝업 표시 중 누르면 팝업이 닫힌다 (취소와 동일)
- [ ] 무한 모드에서 팝업 표시 중 시간이 감소하지 않는다
