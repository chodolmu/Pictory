# S10: 컬렉션 + 업적 (Week 10)

> **Milestone**: M5 — Collection & Economy
> **스프린트 목표**: 컬렉션 3탭(후냐 커스터마이징 / 이마젠 동료 편성 / 플레이어 아이콘) + 업적/칭호 시스템 완성
> **선행 조건**: S09 완료 (이마젠 시스템, 스토리 시스템, 코어 퍼즐, 전체 UI, 13종 기믹, SaveManager)

---

## 설계 개요

### 컬렉션 3탭 구조

```
Tab 1: 후냐 커스터마이징
  - 후냐 캐릭터 외형 변경 (코스튬, 악세서리)
  - 스토리 진행/업적/챕터 클리어로 해금

Tab 2: 이마젠 동료 편성
  - 보유 이마젠 목록, 스킬/스탯 확인
  - 파티 편성 관리 (1~2마리 선택)

Tab 3: 플레이어 아이콘
  - 업적/스토리 진행으로 해금된 아이콘 선택
  - 선택 아이콘 → PlayerProfile 반영 (메인 메뉴 좌상단)
```

### 데이터 흐름

```
스테이지/챕터 클리어 or 업적 달성
  → CollectionManager: 조건 충족 아이템 해금 (커스터마이징, 아이콘)
  → AchievementManager.check_all() → 달성 업적 → 팝업 큐
  → SaveManager.save()
```

## Task 1: CollectionManager

### 설명

컬렉션 전체 상태를 관리하는 Autoload 싱글톤. 후냐 커스터마이징 아이템, 플레이어 아이콘의 해금/적용 상태를 관리한다. 이마젠 편성은 S09의 PartyManager와 연동.

### 수용 기준

- [ ] Autoload `CollectionManager`로 등록
- [ ] 후냐 커스터마이징: `unlock_hunya_item(item_id)`, `equip_hunya_item(item_id)`, `get_equipped_hunya() -> Dictionary`
- [ ] 플레이어 아이콘: `unlock_icon(icon_id)`, `select_icon(icon_id)`, `get_selected_icon() -> String`
- [ ] `get_unlocked_items(category: String) -> Array` — 카테고리별 해금 아이템 목록
- [ ] `signal item_unlocked(category, item_id)` — 아이템 해금 시 시그널
- [ ] SaveManager 연동: `to_save_data()`, `from_save_data()`

### 의존성

- SaveManager (S05), PlayerProfile (S05), PartyManager (S09)

### 구현 노트

- 후냐 아이템 데이터: `resources/collection/hunya_items.json` (id, name, type: costume/accessory, unlock_condition)
- 아이콘 데이터: `resources/collection/player_icons.json` (id, name, unlock_condition)
- 해금 조건 타입: `default`, `chapter_clear`, `achievement`, `story_progress`

### 산출물

- `scripts/collection/collection_manager.gd`
- `resources/collection/hunya_items.json`
- `resources/collection/player_icons.json`

## Task 3: 후냐 커스터마이징 탭

### 설명

후냐 캐릭터의 외형을 변경할 수 있는 탭. 코스튬/악세서리 카테고리로 분류된 아이템을 확인하고 장착한다.

### 수용 기준

- [ ] 후냐 프리뷰 영역: 현재 장착 아이템이 적용된 후냐 표시 (AI-only 도형 기반)
- [ ] 카테고리 전환: `[코스튬]` `[악세서리]`
- [ ] 아이템 그리드: 해금 아이템은 컬러 + 이름, 미해금은 실루엣 + 잠금 아이콘
- [ ] 아이템 선택 → 프리뷰에 즉시 반영
- [ ] `[장착]` 버튼 → CollectionManager에 적용 + SaveManager 저장
- [ ] 현재 장착 중 아이템은 `[장착 중]` 표시

### 의존성

- CollectionManager (Task 1)

### 구현 노트

- 후냐 프리뷰: ColorRect/Polygon2D 조합으로 단순 캐릭터. 코스튬/악세서리는 색상/도형 변경으로 표현
- 아이템 그리드: ScrollContainer > GridContainer (3열)
- 미해금 아이템 `modulate = Color(0.2, 0.2, 0.2, 0.5)`

### 산출물

- `scenes/ui/collection_tab_hunya.tscn`
- `scripts/ui/collection_tab_hunya.gd`

## Task 4: 이마젠 동료 편성 탭

### 설명

보유 이마젠 목록을 확인하고 파티를 편성하는 탭. S09의 PartyManager/이마젠 데이터와 연동.

### 수용 기준

- [ ] 이마젠 그리드: 획득 이마젠은 컬러 + 이름 + 스킬 아이콘, 미획득은 실루엣 + "???"
- [ ] 이마젠 선택 → 상세 패널 (이름, 속성, 스킬 이름/설명/쿨타임, 획득 경로)
- [ ] 파티 편성 슬롯 (상단 1~2칸): 이마젠 드래그 또는 `[편성]` 버튼으로 배치
- [ ] `[편성 해제]` 버튼
- [ ] 편성 결과는 PartyManager에 반영 + SaveManager 저장
- [ ] 이마젠 수집률 표시: "N / M 마리"

### 의존성

- PartyManager (S09), 이마젠 데이터 (S09), CollectionManager (Task 1)

### 구현 노트

- 이마젠 일러스트는 AI-only 도형 (ColorRect + Polygon2D)
- 실루엣: `modulate = Color(0.1, 0.1, 0.1, 1.0)`
- 상세 패널은 별도 씬 또는 팝업
- S09 편성 화면(party_select)과 로직 공유 가능 — 공통 컴포넌트화 고려

### 산출물

- `scenes/ui/collection_tab_imagen.tscn`
- `scripts/ui/collection_tab_imagen.gd`

## Task 5: 플레이어 아이콘 탭

### 설명

플레이어 아이콘을 선택하는 탭. 해금된 아이콘 중 하나를 선택하면 PlayerProfile에 반영되어 메인 메뉴 좌상단에 표시된다.

### 수용 기준

- [ ] 아이콘 그리드: 해금 아이콘은 컬러, 미해금은 실루엣 + 잠금
- [ ] 아이콘 선택 → `[적용]` 버튼 → PlayerProfile.set_icon() + SaveManager 저장
- [ ] 현재 적용 중 아이콘에 체크 표시
- [ ] 해금 조건 표시: "챕터 3 클리어 시 해금" 등

### 의존성

- CollectionManager (Task 1), PlayerProfile (S05)

### 구현 노트

- 아이콘은 AI-only 도형: 원 + 내부 심볼(별, 하트, 크라운 등)을 draw()로 표현
- GridContainer 3~4열
- 적용 시 즉시 메인 메뉴 프로필 위젯도 갱신 (시그널 연결)

### 산출물

- `scenes/ui/collection_tab_icon.tscn`
- `scripts/ui/collection_tab_icon.gd`

업적 달성 시 알림 팝업. 업적 이름, 조건, 보상 표시 + 보상 수령.

### 수용 기준

- [ ] 팝업: `[업적 완료!]` + 업적 이름 + 조건 + 보상
- [ ] `[보상 수령]` → 보상 지급 + 팝업 닫기
- [ ] `[뒤로]` → 미수령 상태로 닫기
- [ ] 동시 달성 업적 큐 → 순차 표시
- [ ] 인게임 중: 상단 배너 간략 알림 → 종료 후 전체 팝업
- [ ] 메인 메뉴 업적 버튼에 미수령 뱃지

### 의존성

- AchievementManager (Task 7), S05 UI 플로우

## Task 7: 도전과제/칭호 시스템

### 설명

업적 조건 정의, 이벤트 기반 달성 체크, 보상(재화/칭호/커스터마이징 아이템/아이콘) 지급.

### 수용 기준

- [ ] Autoload `AchievementManager`
- [ ] 업적 JSON: `resources/collection/achievements.json`
- [ ] 조건 타입: `stage_clear_count`, `chapter_clear`, `total_destroyed`, `max_chain`, `imagen_count`, `infinity_score`, `play_time`, `combo_count`
- [ ] `check_all()`, `check_specific(id)` — 조건 체크
- [ ] `signal achievement_unlocked(id)` — 달성 시그널
- [ ] `grant_reward(id)` — 보상 지급 (재화, 칭호, 커스터마이징 아이템, 아이콘)
- [ ] 칭호: 대표 칭호 설정, 프로필에 표시
- [ ] SaveManager 연동

### 의존성

- SaveManager (S05), CurrencyManager (S05), CollectionManager (Task 1)

### 구현 노트

- 조건 체크: Strategy 패턴 `_check_condition(condition, stats) -> bool`
- 이벤트 기반: `stage_cleared` → 관련 업적만 체크 (성능)
- 보상에서 커스터마이징 아이템/아이콘 해금 시 CollectionManager 호출
- 초기 업적 30~50개

### 산출물

- `scripts/collection/achievement_manager.gd`
- `resources/collection/achievements.json`
